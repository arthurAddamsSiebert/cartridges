Content-Redakteure können im Commerce Management modell-basierte Seitenkomponenten anlegen, bearbeiten und löschen. Für Informationen zu Seitenkomponenten und anderen Content-Management-Konzepten, siehe Content-Management: Begriffe.

Hinweis:

Das Erzeugen von Komponentenmodellen erfordert Entwicklungsaufwände.

Seitenkomponenten können von der Organisation freigegeben werden und auf Channel-Ebene oder Applikationsebene erstellt, bearbeitet etc. werden.

Komponenten Anzeigen

Um alle verfügbaren Komponenten für einen bestimmten Kontext anzuzeigen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox.

    Die Navigationsleiste für den Managementkontext wird geöffnet.

  2. Klicken Sie in der Navigationsleiste auf Content | Komponenten.

    Die Komponenten-Listenansicht wird angezeigt.

  3. Wählen Sie den Applikationstyp aus (optional).

    Im Channel- oder Organisationskontext zeigt die Liste alle Komponenten aller im jeweiligen Kontext verfügbaren Applikationstypen an. Mit der Auswahl eines Applikationstyps legen Sie das Content-Modell, mit dem Sie arbeiten werden, fest (siehe Was ist eine Applikation?) und filtern die Komponenten entsprechend.

    Abbildung 39: Auswahl des Content-Modells
    Tip:

    Diese Einstellung ist während Ihrer gesamten Commerce Management-Sitzung aktiv. Um die Auswahl aufzuheben, wählen Sie einen anderen Applikationstyp oder "Alle" aus.

Für eine Einführung in das Modul Komponenten, siehe Modul Komponenten.

Verfügbare Komponententypen anzeigen

Commerce Management kann eine Liste der Komponententypen mit Parametern, Seitenbereichen usw. anzeigen, die in einem gegebenen Management-Kontext verfügbar sind.

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Komponenten.

    Die Komponenten-Listenansicht wird angezeigt.

  2. Klcken Sie oben in der Komponenten-Listenansicht auf Komponententyp-Überblick.

    Das löst ein Skript aus, das alle relevanten Informationen über die zur Zeit im gegebenen Management-Kontext verfügbaren Komponententypen zusammenfasst, und diese Information in einem neuen Browser-Fenster anzeigt.

Komponenten suchen

Einfache Komponentensuche

In der einfachen Suche lassen sich Komponenten nach Namen oder Teilen des Inhalts suchen. Falls erforderlich, können Sie die Suche auf einen bestimmten Komponententyp oder eine Gruppe eingrenzen.

Um die einfache Suche zu verwenden:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Komponenten.

    Die Komponenten-Listenansicht wird angezeigt.

  2. Geben Sie den Namen der Komponente oder einen Teil des Inhalts als Suchbegriff ein.

    Falls erforderlich, wählen Sie einen Komponententyp aus, um die Suche einzugrenzen.

    Hinweis:

    Es ist nicht möglich, nach Content von freigegebenen Komponenten zu suchen, deren Inhalt überschrieben wurde, siehe Steuern von Änderungen an freigegebenem Content. Die Suche liefert nur Treffer aus dem Original-Content.

  3. Klicken Sie auf Suchen.

    Die Suchergebnisse werden in der Listenansicht angezeigt.

Tip:

Standardmäßig wird die einfache Suche angezeigt. Wenn Sie zuvor die erweiterte Suche verwendet haben, klicken Sie auf Einfache Suche.

Erweiterte Komponentensuche

Mit der erweiterten Suche können Sie durch die Angabe exakter Suchkriterien gezielter suchen. Um die erweiterte Suchmaske zu verwenden:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Komponenten.

    Die Komponenten-Listenansicht wird angezeigt. Standardmäßig wird die einfache Suche angezeigt.

  2. Klicken Sie auf Erweiterte Suche.

  3. Geben Sie Ihre Suchkriterien in den entsprechenden Feldern an.

    Die Felder sind selbsterklärend.

    Hinweis:

    Es ist nicht möglich, nach Content von freigegebenen Komponenten zu suchen, deren Inhalt überschrieben wurde, siehe Steuern von Änderungen an freigegebenem Content. Die Suche liefert nur Treffer aus dem Original-Content.

    Tip:

    Seiten- oder Komponentenfilter schränken die Suche auf Komponenten ein, die den damit herausgefilterten Seiten oder Komponenten zugeordnet sind. Das Anlegen von Filtern stellt eine zusätzliche erweiterte Suche nach dem entsprechenden Objekt dar. Sobald der Filter angelegt und angewendet ist, kehren Sie zur ursprünglichen erweiterten Komponentensuche zurück.

  4. Klicken Sie auf Suchen.

    Die Suchergebnisse werden in der Listenansicht angezeigt.

Komponenten erstellen

Um eine neue Komponente anzulegen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Komponenten.

    Die Komponenten-Listenansicht wird angezeigt.

  2. Klicken Sie Neu.

    Der Dialog Neue Komponente wird angezeigt.

  3. Geben Sie einen Namen für die neue Komponente an.

    Falls erforderlich, modifizieren Sie die automatisch erzeugte ID.

  4. Wählen Sie ein Komponenten-Template aus.

    Mit einem Klick auf das Feld zur Dateiauswahl wird ein Dialog zur Auswahl des Komponenten-Templates geöffnet.

    Tip:

    Ein Klick auf den Komponententyp-Überblick im Template-Auswahldialog öffnet eine Liste mit Komponententypen mit Parametern, Slots usw., die in einem bestimmten Management-Kontext verfügbar sind.

  5. Legen Sie Publikations-Status und -Dauer fest.

  6. Klicken Sie Übernehmen.

    Die neue Komponente wird angelegt und in der Detailansicht zur Bearbeitung geöffnet, siehe Komponenten bearbeiten.

Komponenten sperren

Bevor Sie eine Komponente bearbeiten, sollten Sie sie sperren. Die Sperre verhindert, dass die Komponente zur gleichen Zeit von anderen Benutzern bearbeitet werden kann. Die Komponentensperre kann manuell gelöst werden. Nach Ablauf eines festgelegten Intervalls wird die Produktsperre automatisch aufgehoben. Information zum Konfigurieren der Komponentensperre, siehe Zeitraum für Content-Sperre festlegen.

Tip:

Beim Bearbeiten einer Komponente wird die Komponente automatisch für Sie gesperrt, falls sie nicht schon vorher von einem anderen Benutzer gesperrt wurde.

Komponenten können auf verschiedene Weise gesperrt werden: entweder direkt über der Komponentenliste oder über die Komponenten-Detailansicht.

Sperren von Komponenten in der Listenansicht

Um Komponenten über die Komponenten-Listenansicht zu sperren:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Komponenten.

    Die Komponenten-Listenansicht wird angezeigt.

  2. Klicken Sie auf das Symbol in der Zeile der Komponente(n), die Sie bearbeiten wollen.

    Falls die Komponente gegenwärtig von einem anderen Benutzer bearbeitet wird, erscheint eine Fehlermeldung.

Hinweis:

Klicken Sie auf das Symbol , um die Komponentensperre aufzuheben, wenn Sie mit der Bearbeitung fertig sind.

Sperren von Komponenten in der Detailansicht

Um eine Komponente über die Komponenten-Detailansicht zu sperren:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Komponenten.

    Die Komponenten-Listenansicht wird angezeigt.

  2. Klicken Sie auf den Namen der Komponente, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Klicken Sie auf das Symbol oben im Dialog.

    Falls die Komponente gegenwärtig von einem anderen Benutzer bearbeitet wird, erscheint eine Fehlermeldung.

Hinweis:

Klicken Sie auf das Symbol , um die Komponentensperre aufzuheben, wenn Sie mit der Bearbeitung fertig sind.

Komponenten bearbeiten

Zu den Bearbeitungsfunktionen für Komponenten gehören das Bearbeiten von Komponenteninhalten und das Kopieren von Komponenten. Vor dem Bearbeiten müssen die Komponenten gesperrt werden, siehe Komponenten sperren.

Komponenteninhalt bearbeiten

Um den Inhalt von Komponenten zu bearbeiten:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Komponenten.

    Die Komponenten-Listenansicht wird angezeigt.

  2. Klicken Sie auf den Namen der Komponente, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Legen Sie die erforderlichen Parameter-Werte fest und bearbeiten Sie den Content wie erforderlich.

    Die verfügbaren Parameter oder Content-Elemente hängen vom Komponenten-Template ab.

    Tip:

    Parameterwerte, die aus Stammobjekten vererbt wurden, können bearbeitet werden, siehe Ererbte Parameter bearbeiten.

  4. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

    Anderenfalls klicken Sie auf Zurücksetzen, um die Änderungen zu verwerfen.

Standard-Komponenten-Content verbergen

Komponenten-Templates (siehe Verwaltung von Komponenten-Templates) können an abgeleitete Komponenten Platzhalter mit bereits zugewiesenem Standard-Content weitergeben. Beim Bearbeiten der Komponenten können Sie diesen Standard-Content ausblenden. Um dies zu tun:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Komponenten.

    Die Komponenten-Listenansicht wird angezeigt.

  2. Klicken Sie auf den Namen der Komponente, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Klicken Sie im Abschnitt Platzhalter auf das Symbol neben dem betreffenden Platzhalter.

    Der Dialog Einstellungen wird angezeigt.

  4. Markieren Sie das Kontrollkästchen Standardcontent ausblenden, und klicken Sie auf Übernehmen.

    Der vom Template abgeleitete Content wird nicht mehr angezeigt.

    Tip:

    Bei Komponenten, die aus einem Stamm-Management-Kontext freigegeben wurden (siehe Freigaben von Content-Elementen), ist die Option, den Standard-Content zu verbergen, nicht verfügbar.

Platzierung von Komponenten-Content steuern

Komponenten-Templates (siehe Verwaltung von Komponenten-Templates) können an abgeleitete Komponenten Platzhalter mit bereits zugewiesenem Standard-Content weitergeben. Wenn Sie eigenen Content in diese Platzhalter einfügen, wird dieser normalerweise unter dem Standard-Content positioniert. Sie können dieses Verhalten jedoch ändern und eigenen Content oberhalb des Standard-Contents anzeigen lassen. Um dies zu tun:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Komponenten.

    Die Komponenten-Listenansicht wird angezeigt.

  2. Klicken Sie auf den Namen der Komponente, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Klicken Sie im Abschnitt Platzhalter auf das Symbol neben dem betreffenden Platzhalter.

    Der Dialog Einstellungen wird angezeigt.

  4. Markieren Sie das Kontrollkästchen Komponenten über Standard-Content platzieren und klicken Sie auf Übernehmen.

    Lokal zugewiesene Komponenten werden jetzt oberhalb des Standard-Contents platziert.

    Tip:

    Bei Komponenten, die aus einem Stamm-Management-Kontext freigegeben wurden (siehe Freigaben von Content-Elementen), ist die Option, eigene Komponenten über dem Standard-Content anzuzeigen, nicht verfügbar.

Ererbte Parameter bearbeiten

Es werden standardmäßig alle Parameterwerte aus den Stammobjekten vererbt. Jedoch können sie auf jeder Vererbungs-Ebene überladen sein. Überladene Werte können auf den ursprünglichen (ererbten) Wert entsprechend zurückgesetzt werden.

Ererbte Parameter Überladen

Um einen ererbten Parameterwert zu überladen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Komponenten.

    Die Komponenten-Listenansicht wird angezeigt.

  2. Klicken Sie auf den Namen der Komponente, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Beginnen Sie im Abschnitt Parameter mit der Bearbeitung der gewünschten Parameterwerte.

    Wird ein Wert bearbeitet, so zeigt ein Stift-Symbol () neben dem Wert an, dass dieser Wert nicht mehr ererbt, sondern überladen ist.

  4. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

    Anderenfalls klicken Sie auf Zurücksetzen, um die Änderungen zu verwerfen.

Überladen von Parametern Zurücksetzen

Um einen überladenen Parameter auf seine ursprünglichen (ererbten) Wert zurückzusetzen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Komponenten.

    Die Komponenten-Listenansicht wird angezeigt.

  2. Klicken Sie auf den Namen der Komponente, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Klicken Sie im Abschnitt Parameter auf Zurücksetzen auf geerbte Werte.

    Es werden Kontrollkästchen vor den bearbeiteten Parametern angezeigt.

  4. Wählen Sie die Parameter, die zurückgesetzt werden sollen.

    Verwenden Sie die Kontrollkästchen, um die gewünschten Parameter auszuwählen.

  5. Klicken Sie auf Zurücksetzen.

    Die ausgewählten Parameter werden auf ihre ursprünglichen (ererbten) Werte zurückgesetzt.

Zugewiesene Objekte verwalten

Komponenten können verschiedenen Business-Objekten zugewiesen werden. Das beinhaltet:

  • Kundensegmente

    Sie können mehrere Kundensegmente hinzufügen. Wenn Sie das tun, können Sie keine Aktionen oder Kampagnen zuordnen.

  • Aktionen

    Sie können eine Aktion zuordnen. Wenn Sie das tun, können Sie keine Kundensegmente hinzufügen oder Kampagnen zuordnen.

  • Kampagnen

    Sie können eine Kampagne zuordnen. Wenn Sie das tun, können Sie keine Kundensegmente hinzufügen oder Aktionen zuordnen.

  • A/B-Tests

    Sie können eine oder mehrere Testgruppen aus (mehreren) A/B-Tests hinzufügen. Davon werden die anderen Business-Objekt-Beziehungen nicht beeinträchtigt.

Erzeugen von Business-Objekt-Beziehungen

Um Business-Objekt-Beziehungen für Komponenten zu erzeugen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Komponenten.

    Die Komponenten-Listenansicht wird angezeigt.

  2. Klicken Sie auf den Namen der Komponente, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Wechseln Sie zum Register Eigenschaften.

    Das Register Eigenschaften sieht u.a. vor, Business-Objekt-Beziehungen zu verwalten.

  4. Die gewünschten Business-Objekte hinzufügen bzw. zuordnen.

    Klicken Sie Hinzufügen/Zuordnen, um einen Dialog zu öffnen und die gewünschten Business-Objekte auszuwählen und hinzuzufügen.

  5. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

    Anderenfalls klicken Sie auf Zurücksetzen, um die Änderungen zu verwerfen.

Entfernen von Objekt-Beziehungen

Um Business-Objekt-Beziehungen für Komponenten zu entfernen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Komponenten.

    Die Komponenten-Listenansicht wird angezeigt.

  2. Klicken Sie auf den Namen der Komponente, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Wechseln Sie zum Register Eigenschaften.

    Das Register Eigenschaften sieht u.a. vor, Business-Objekt-Beziehungen zu verwalten.

  4. Entfernen Sie Business-Objekt-Beziehungen wie erforderlich.

    Um eine Zuordnung zu einem Kundensegment oder einer A/B-Testgruppe zu entfernen, wählen Sie die entsprechende Zuordnung und klicken Sie Entfernen.

    Um eine Zuordnung zu einer Aktion oder Kampagne zu entfernen, klicken Sie neben der entsprechenden Zuordnung auf Aufheben.

  5. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

    Anderenfalls klicken Sie auf Zurücksetzen, um die Änderungen zu verwerfen.

Komponenten kopieren

Beim Kopieren von Komponenten werden alle dazugehörigen Content-Elemente ebenfalls kopiert, dagegen werden Zuordnungen zu Seiten, Seitenvarianten, Seiten-Templates, Komponenten-Templates oder anderen Komponenten nicht mit kopiert. Um eine Komponente zu kopieren:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Komponenten.

    Die Komponenten-Listenansicht wird angezeigt.

  2. Klicken Sie auf das Symbol in der Zeile der Komponente(n), die Sie kopieren wollen.

    Der Dialog Komponente kopieren wird angezeigt.

  3. Bearbeiten Sie Namen, ID und Publikationszeitraum wie erforderlich.

  4. Klicken Sie Übernehmen.

    Die Kopie der Komponente wird angelegt und in der Detailansicht geöffnet.

Komponenten-Zuordnungen verwalten

Das Verwalten von Komponenten-Zuordnungen bedeutet, dass Komponenten zu Seitenvarianten, Komponenten, Includes, Seiten-Templates oder Komponenten-Templates hinzugefügt werden (wie vom Komponententyp zulässig) und die Zuordnungen entfernt werden können.

Komponenten zu Zielobjekten zuordnen

Es gibt zwei Arten, um eine bestehende Komponente einem bestehenden Zielobjekt zuzuordnen: entweder beginnen Sie, wie unten beschrieben, bei Komponenten über das Register Zuordnungen, oder Sie beginnen beim Zielobjekt über das Register Content-Ansicht.

Hinweis:

Nur publizierte Komponenten können online sichtbar sein, unabhängig von ihren Seitenbereich-Zuordnungen.

Um eine Ziel-Zuordnung für eine Komponente anzulegen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Komponenten.

    Die Komponenten-Listenansicht wird angezeigt.

  2. Klicken Sie auf den Namen der Komponente, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Öffnen Sie das Register Zuordnungen.

    Für jeden verfügbaren Objekttyp gibt es einen Abschnitt. Abschnitte mit vorhandenen Zuordnungen werden aufgeklappt, andere zugeklappt.

    Hinweis:

    Die verfügbaren Zielobjekte hängen von den zulässigen Zuordnungen ab, je nach Komponententyp.

  4. Klicken Sie im entsprechenden Abschnitt auf Zuordnen.

    Der Assistent zum Zuordnen der aktuellen Komponente zu einem bestehenden Zielobjekt des gewählten Typs wird geöffnet.

  5. Markieren Sie das gewünschte Zielobjekt und klicken Sie auf Weiter.

  6. Legen Sie den Sichtbarkeitszeitraum und die Priorität fest.

  7. Markieren Sie einen Seitenbereich, dem die Komponente zugeordnet werden soll, und klicken Sie auf Fertigstellen.

    Die Komponente wird dem gewählten Zielobjekt zugeordnet.

Zuordnungen von Komponenten aufheben

Um die Zuordnung einer Komponente zu einem Zielobjekt aufzuheben:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Komponenten.

    Die Komponenten-Listenansicht wird angezeigt.

  2. Klicken Sie auf den Namen der Komponente, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Öffnen Sie das Register Zuordnungen.

  4. Markieren Sie Zuordnung, die Sie aufheben wollen.

    Markieren Sie das Kontrollkästchen der gewählten Zuordnung im entsprechenden Abschnitt.

  5. Klicken Sie auf Aufheben und bestätigen Sie die Operation.

    Die Zuordnung der Komponente zum aktuellen Objekt wird entfernt.

Komponenten-Publikationseinstellungen festlegen

Publikation bezieht sich auf die allgemeine Verfügbarkeit der Komponente. Nur publizierte Komponenten können sichtbar sein, d.h. online gesetzt werden. Damit Komponenten in der Storefront sichtbar werden, müssen die Publikationsdauer sowie der Sichtbarkeitszeitraum und die Priorität übereinstimmen.

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Komponenten.

    Die Komponenten-Listenansicht wird angezeigt.

  2. Klicken Sie auf das Publikationsdatum der Komponente, die Sie bearbeiten wollen.

    Das Register Eigenschaften der Detailansicht wird geöffnet.

  3. Legen Sie Publikations-Status und -Dauer fest wie erforderlich.

    Markieren Sie das Optionsfeld des gewünschten Status und geben Sie die Publiziert-von- und Publiziert-bis-Daten und -Zeiten an.

    Hinweis:

    Leeres Enddatum und -zeit setzen die Publikationsdauer auf "unbegrenzt".

  4. Klicken Sie Übernehmen.

    Die Einstellungen für Publikations-Status und -Dauer werden sofort aktiv.

Die Sichtbarkeit einer Komponente, d.h. die Verfügbarkeit in einem Seitenbereich, wenn sie explizit zugeordnet wurde, wird über die Verwaltung von Seitenvarianten gesteuert. Für Informationen zum Einstellen der Sichtbarkeit, siehe Komponenten-Sichtbarkeit festlegen.

Komponenten-Label verwalten

Label ermöglichen die Definition von Gruppen abhängiger Objekte, um dann bestimmte Aktionen mit diesen Objekten auszuführen. Informationen zum Erzeugen von Labeln, siehe Label anlegen. Für weitere Informationen zur Ausführung label-basierter Aktionen, siehe Aktionen für Objekte mit Labeln durchführen.

Label zu Komponenten hinzufügen

Um einer Komponente ein Label zuzuordnen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Komponenten.

    Die Komponenten-Listenansicht wird angezeigt.

  2. Klicken Sie auf den Namen der Komponente, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Öffnen Sie das Register Label.

    Eine Liste der Label, die dieser Komponente zugeordnet sind, wird angezeigt.

  4. Klicken Sie auf Neu, um eine neue Label-Zuordnung zu definieren.

    Eine Liste der verfügbaren Label wird angezeigt.

  5. Markieren Sie das gewünschte Label und klicken Sie auf OK.

    Das ausgewählte Label wird der aktuellen Komponente zugeordnet.

Label von Komponenten entfernen

Um die Zuordnung eines Labels zu einer Komponente aufzuheben:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Komponenten.

    Die Komponenten-Listenansicht wird angezeigt.

  2. Klicken Sie auf den Namen der Komponente, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Öffnen Sie das Register Label.

    Eine Liste der Label, die dieser Komponente zugeordnet sind, wird angezeigt.

  4. Markieren Sie das Kontrollkästchen der Labelzuordnung, die Sie aufheben wollen.

  5. Klicken Sie auf Aufheben und bestätigen Sie die Operation.

    Die Zuordnung des Labels zur aktuellen Komponente wird entfernt.

Komponenten in Stapelprozessen bearbeiten

Bestimmte Operationen in der Komponentenverwaltung können mittels Stapelprozessen mit mehreren Komponenten gleichzeitig ausgeführt werden. Operationen, die als Stapelprozesse ausgeführt werden können, sind:

  • Publikationszeitraum der ausgewählten Komponenten festlegen

  • Ausgewählte Komponenten zu Seitenvarianten zuordnen

  • Ausgewählte Komponenten zu Includes zuordnen

  • Ausgewählte Komponenten Komponenten zuordnen

  • Ausgewählten Komponenten Komponente zuordnen

  • Ausgewählten Komponenten Label zuordnen

  • Markierte Seitenkomponenten löschen

Komponenten publizieren

In einer Umgebung der Datenreplizierung müssen Komponenten, die im Bearbeitungssystem erzeugt werden, im Zielsystem veröffentlicht werden. Dies geschieht entweder automatisch im Kontext von regulärer Datenreplizierung (siehe Verwaltung von Replikationsaufgaben) oder manuell, wie unten beschrieben.

Um Komponenten manuell zu publizieren, gehen Sie wie folgt vor:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Komponenten.

    Die Komponenten-Listenansicht wird angezeigt.

  2. Markieren Sie die Kontrollkästchen der zu publizierenden Komponenten in der Listenansicht.

    Alternativ können Sie eine Suche ausführen und auf Alles auswählen klicken, um das Suchergebnis auszuwählen.

  3. Klicken Sie Publizieren im Zielsystem.

    Die Liste konfigurierter Zielsysteme wird angezeigt.

  4. Wählen Sie das gewünschte Zielsystem und die Lösung für einen Sperrkonflikt.

    Für Informationen zu den Sperrkonflikt-Lösungen, siehe Objektsperren.

  5. Klicken Sie auf Fertigstellen.

    Die gewählten Komponenten werden an das gewählte Zielsystem repliziert.

Verwalten von Komponenten per Drag & Drop

Ab der Version 7.9 verfügt die Design-Ansicht über einen Layout-Modus, um auf intuitivere Art Komponenten zu verwalten.

Im Gegensatz zur Daten-Ansicht, bei der die Erzeugung und Einbindung einer Komponente klar unterscheidbare Schritte sind, geschieht dies bei der Benutzung der Layout-Ansicht in einem Arbeitschritt. Bitte beachten Sie, dass eine so erstellte Komponente nicht gesperrt wird und sofort publiziert wird. Die Einstellungen für Sperrung und Publikation können in der Daten-Ansicht vorgenommen werden. Für weitere Informationen, siehe Komponenten sperren und Komponenten-Publikationseinstellungen festlegen.

Hinweis:

Die Palette enthält alle Komponent-Templates sowie alle Medien-Komponenten. Sollten Sie versuchen eine Komponente zu einem unpassendem Seitenbereich oder Platzhalter hinzuzufügen, erfolgt eine Fehlermeldung und die Komponente wird nicht erzeugt.

Tip:

Eine neue Komponente wird als letztes Element zu einem Seitenbereich hinzugefügt. Komponenten werden abhängig von ihrer Priorität und der Multiplizität des Seitenbereichs angezeigt. Das heißt, kann ein Seitenbereich n Komponenten anzeigen, ihre ist jedoch die n+1.e, wird die Komponente nicht gerendert. Um die neue Komponente anzuzeigen, wählen sie den Seitenbereich aus und ändern Sie die Priorität der Komponenten-Zuweisung.

Erstellen von Komponenten per Drag & Drop

  1. Öffnen Sie das Register Komponenten-Templates.

  2. Ziehen Sie eine Kachel von der Palette zu einem Seitenbereich oder Platzhalter.

    Eine neue Komponente basierend auf dem ausgewählten Template wird erzeugt und dem entsprechenden Bereich zugewiesen. Die neue Komponente ist ausgewählt in der Baumansicht und im Bearbeitungsbereich werden die Eigenschaften angezeigt.

  3. Wechseln Sie zum Bearbeitungsbereich und konfigurieren Sie die neue Medien-Komponente.
