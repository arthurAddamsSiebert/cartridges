Content-Redakteure können im Commerce Management Ansichtskontexte anlegen, bearbeiten und löschen. Für Informationen zu Ansichtskontexten und anderen Content-Management-Konzepten, siehe Content-Management: Begriffe.

Ansichtskontexte können von der Organisation freigegeben werden und auf Channel-Ebene oder Applikationsebene erstellt, bearbeitet etc. werden.

Ansichtskontexte Anzeigen

Um alle verfügbaren Ansichts-Kontexte für einen bestimmten Kontext anzuzeigen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox.

    Die Navigationsleiste für den Managementkontext wird geöffnet.

  2. Klicken Sie in der Navigationsleiste auf Content | Ansichts-Kontexte.

    Die Listenansicht für Ansichtskontexte wird angezeigt.

  3. Wählen Sie den Applikationstyp aus (optional).

    Im Channel- oder Organisationskontext zeigt die Liste alle Ansichtskontexte aller im jeweiligen Kontext verfügbaren Applikationstypen an. Mit der Auswahl eines Applikationstyps legen Sie das Content-Modell, mit dem Sie arbeiten werden, fest (siehe Was ist eine Applikation?) und filtern die Ansichtskontexte entsprechend.

    Abbildung 42: Auswahl des Content-Modells
    Tip:

    Diese Einstellung ist während Ihrer gesamten Commerce Management-Sitzung aktiv. Um die Auswahl aufzuheben, wählen Sie einen anderen Applikationstyp oder "Alle" aus.

Für eine Einführung in das Modul Ansichtskontexte, siehe Modul Ansichtskontext.

Ansichtskontexte suchen

In der Suchmaske der Listenansicht lassen sich Ansichtskontexte nach Name oder ID suchen. Sie können die Suche auf eine bestimmte Gruppe oder einen bestimmten Typ von Ansichtskontexten eingrenzen. In der erweiterten Suche lässt sich, falls erforderlich, gezielt nach Ansichtskontexten suchen, die ausschließlich Seiten oder Includes zugeordnet sind.

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Ansichts-Kontexte.

    Die Listenansicht für Ansichtskontexte wird angezeigt.

  2. Geben Sie den Namen des Ansichtskontexts oder die ID als Suchbegriff ein.

    Falls erforderlich, können Sie die Suche durch Auswahl eines Ansichtskontext-Typs und/oder Zuordnungen ausschließlich zu Seiten/Includes eingrenzen.

  3. Klicken Sie auf Suchen.

    Die Suchergebnisse werden in der Listenansicht angezeigt.

Ansichtskontexte erstellen

Um einen neuen Ansichtskontext anzulegen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Ansichts-Kontexte.

    Die Listenansicht für Ansichtskontexte wird angezeigt.

  2. Klicken Sie auf Neu.

    Der Dialog Neuer Ansichtskontext wird angezeigt.

  3. Legen Sie die Sprache fest, für die der Ansichtskontext angelegt werden soll.

    Wählen Sie die gewünschte Locale aus der Dropdown-Liste aus und klicken Sie auf Übernehmen.

  4. Geben Sie einen Namen und eine ID für den neuen Ansichtskontext an.

    Falls erforderlich, geben Sie eine Beschreibung an.

  5. Bestimmen Sie den Typ des Ansichtskontexts.

    Wählen Sie aus der Dropdown-Liste den gewünschten Typ aus.

  6. Klicken Sie auf Übernehmen.

    Der neue Ansichtskontext wird angelegt und in der Detailansicht zur Bearbeitung geöffnet, siehe Ansichtskontexte bearbeiten.

Ansichtskontexte sperren

Bevor Sie einen Ansichtskontext bearbeiten, sollten Sie diesen sperren. Die Sperre verhindert, dass der Ansichtskontext zur gleichen Zeit von anderen Benutzern bearbeitet werden kann. Die Sperre kann manuell gelöst werden. Nach Ablauf eines festgelegten Intervalls wird die Produktsperre automatisch aufgehoben. Information zum Konfigurieren der Sperre, siehe Zeitraum für Content-Sperre festlegen.

Tip:

Beim Bearbeiten eines Ansichts-Kontextes wird dieser automatisch für Sie gesperrt, falls sie nicht schon vorher von einem anderen Benutzer gesperrt wurde.

Um einen Ansichtskontext über die Detailansicht zu sperren:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Ansichts-Kontexte.

    Die Listenansicht für Ansichtskontexte wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID des Ansichtskontextes, den Sie bearbeiten wollen.

    Das Register Details der Detailansicht wird geöffnet.

  3. Klicken Sie auf das Symbol oben im Dialog.

    Falls der Ansichtskontext gegenwärtig von einem anderen Benutzer bearbeitet wird, erscheint eine Fehlermeldung.

Hinweis:

Klicken Sie auf das Symbol um die Sperre aufzuheben, wenn Sie mit der Bearbeitung fertig sind.

Ansichtskontexte bearbeiten

Allgemeine Optionen zur Bearbeitung von Ansichtskontexten sind das Bearbeiten von Details und von Einstellungen, wie z. B. Name und Beschreibung. Bearbeitbare Details sind Parameter (abhängig vom Typ) und Content von Elementen, die als "globale Fallbacks" definiert werden. ID und Typ von bestehenden Ansichtskontexten können nicht verändert werden.

Vor dem Bearbeiten müssen die Ansichtskontexte gesperrt werden, siehe Ansichtskontexte sperren.

Details von Ansichtskontexten bearbeiten

Um Details von Ansichtskontexten zu bearbeiten:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Ansichts-Kontexte.

    Die Listenansicht für Ansichtskontexte wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID des Ansichtskontextes, den Sie bearbeiten wollen.

    Das Register Details der Detailansicht wird geöffnet.

  3. Legen Sie die erforderlichen Parameter-Werte fest.

    Die verfügbaren Parameter hängen vom Typ des Ansichtskontextes ab.

    Hinweis:

    Globale Fallback-Optionen können nicht verändert werden. Sie können aber das Fallback-Element öffnen und dessen Content bearbeiten.

  4. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

    Anderenfalls klicken Sie auf Zurücksetzen, um die Änderungen zu verwerfen.

Eigenschaften von Ansichtskontexten bearbeiten

Um Eigenschaften von Ansichtskontexten zu bearbeiten:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Ansichts-Kontexte.

    Die Listenansicht für Ansichtskontexte wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID des Ansichtskontextes, den Sie bearbeiten wollen.

    Das Register Details der Detailansicht wird geöffnet.

  3. Wechseln Sie zum Register Eigenschaften.

    Das Register Eigenschaften ermöglicht die Bearbeitung des Anzeigennamens und der Beschreibung einer gewählten Locale.

  4. Bearbeiten Sie die Eigenschaften wie erforderlich.

  5. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

    Anderenfalls klicken Sie auf Zurücksetzen, um die Änderungen zu verwerfen.

Ansichtskontext-Zuordnungen verwalten

Die Verwaltung von Ansichtskontext-Zuordnungen besteht aus

  • der Überprüfung der Zuordnungen von Seiten oder Includes (je nach Ansichtskontext-Typ) zu Business-Objekten wie Produkten oder Kategorien,

  • dem Löschen von Seiten oder Includes.

Hinweis:

Das Erzeugen von Zuordnungen zwischen Business-Objekten und Seiten/Includes ist Aufgabe der Produktverwaltung bzw. der Katalogverwaltung. Für weitere Informationen, siehe Produkt-Content-Ansichten verwalten oder Katalog-Content-Ansichten verwalten.

Überprüfung von Ansichtskontext-Zuordnungen

Um einen Überblick über die bestehenden Zuordnungen zwischen Business-Objekten und den Seiten/Includes zu erhalten:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Ansichts-Kontexte.

    Die Listenansicht für Ansichtskontexte wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID des Ansichtskontextes, den Sie bearbeiten wollen.

    Das Register Details der Detailansicht wird geöffnet.

  3. Wechseln Sie zum Register Seiten bzw. zum Register Includes.

    Je nach Typ des Ansichtskontexts listet das Register alle Seiten oder Includes auf, die Produkten und Kategorien zugeordnet werden.

Tip:

Durch Klicken auf den Namen von Produkt/Kategorie oder Seite/Include wird die entsprechende Detailansicht geöffnet.

Zugeordnete Seiten/Includes löschen

Um eine Seite oder Include zu löschen, die einem Produkt/einer Kategorie zugeordnet ist:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Ansichts-Kontexte.

    Die Listenansicht für Ansichtskontexte wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID des Ansichtskontextes, den Sie bearbeiten wollen.

    Das Register Details der Detailansicht wird geöffnet.

  3. Wechseln Sie zum Register Seiten bzw. zum Register Includes.

    Je nach Typ des Ansichtskontexts listet das Register alle Seiten oder Includes auf, die Produkten und Kategorien zugeordnet werden.

  4. Wählen Sie die Zuordnung, von der Sie die Seite oder Include entfernen wollen.

    Markieren Sie das Kontrollkästchen der gewählten Zuordnung in der entsprechenden Zeile.

  5. Klicken Sie auf Löschen und bestätigen Sie die Operation.

    Die Seite oder Include wird aus dem System entfernt.

Ansichtskontexte publizieren

In einer Datenreplikations-Umgebung müssen Ansichtskontexte, die im Bearbeitungssystem erzeugt werden, im Zielsystem veröffentlicht werden. Dies geschieht entweder automatisch im Kontext von regulärer Datenreplizierung (siehe Verwaltung von Replikationsaufgaben) oder manuell, wie unten beschrieben.

Um Ansichtskontexte manuell zu publizieren, gehen Sie wie folgt vor:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Ansichts-Kontexte.

    Die Listenansicht für Ansichtskontexte wird angezeigt.

  2. Markieren Sie die Kontrollkästchen der zu publizierenden Ansichtskontexte in der Listenansicht.

    Alternativ können Sie eine Suche ausführen und auf Alles auswählen klicken, um das Suchergebnis auszuwählen.

  3. Klicken Sie Publizieren im Zielsystem.

    Die Liste konfigurierter Zielsysteme wird angezeigt.

  4. Wählen Sie das gewünschte Zielsystem und die Lösung für einen Sperrkonflikt.

    Für Informationen zu den Sperrkonflikt-Lösungen, siehe Objektsperren.

  5. Klicken Sie auf Fertigstellen.

    Die gewählten Ansichtskontexte werden an das gewählte Zielsystem repliziert.
