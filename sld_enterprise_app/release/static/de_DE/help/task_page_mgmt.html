Content-Redakteure können im Commerce Management neue Seiten anlegen und bestehende Seiten bearbeiten. Für Informationen zu Seiten und anderen Content-Management-Konzepten, siehe Content-Management: Begriffe.

Seiten können von der Organisation freigegeben werden und auf Channel-Ebene oder Applikationsebene erstellt, bearbeitet etc. werden.

Seiten anzeigen

Um alle verfügbaren Seiten für einen bestimmten Kontext anzuzeigen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox.

    Die Navigationsleiste für den Managementkontext wird geöffnet.

  2. Klicken Sie in der Navigationsleiste auf Content | Seiten.

    Die Seiten-Baumansicht wird angezeigt.

  3. Wählen Sie den Applikationstyp aus (optional).

    Im Channel- oder Organisationskontext zeigt die Liste alle Seiten aller im jeweiligen Kontext verfügbaren Applikationstypen an. Mit der Auswahl eines Applikationstyps legen Sie das Content-Modell, mit dem Sie arbeiten werden, fest (siehe Was ist eine Applikation?) und filtern die Seiten entsprechend.

    Abbildung 37: Auswahl des Content-Modells
    Tip:

    Diese Einstellung ist während Ihrer gesamten Commerce Management-Sitzung aktiv. Um die Auswahl aufzuheben, wählen Sie einen anderen Applikationstyp oder "Alle" aus.

Für eine Einführung in das Modul Seiten, siehe Modul Seiten.

Seiten suchen

Die Seitensuche ist über die Listenansicht der Seiten verfügbar.

Einfache Seitensuche

In der einfachen Suche lassen sich Seiten nach Namen oder ID suchen. Falls erforderlich, können Sie die Suche auf eine bestimmte Seitengruppe oder einen Seitentyp eingrenzen.

Um die einfache Suche zu verwenden:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten.

    Die Seiten-Baumansicht wird angezeigt.

  2. Öffnen Sie das Register Listenansicht.

  3. Geben Sie den Namen der Seite oder die ID als Suchbegriff ein.

    Falls erforderlich, wählen Sie einen Typ aus, um die Suche einzugrenzen.

  4. Klicken Sie auf Suchen.

    Die Suchergebnisse werden in der Listenansicht angezeigt.

Tip:

Standardmäßig wird die einfache Suche angezeigt. Wenn Sie zuvor die erweiterte Suche verwendet haben, klicken Sie auf Einfache Suche.

Erweiterte Seitensuche

Mit der erweiterten Suche können Sie durch die Angabe exakter Suchkriterien gezielter suchen. Um die erweiterte Suchmaske zu verwenden:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten.

    Die Seiten-Baumansicht wird angezeigt.

  2. Öffnen Sie das Register Listenansicht.

    Standardmäßig wird die einfache Suche angezeigt.

  3. Klicken Sie auf Erweiterte Suche.

  4. Geben Sie Ihre Suchkriterien in den entsprechenden Feldern an.

    Die Felder sind selbsterklärend.

  5. Klicken Sie auf Suchen.

    Die Suchergebnisse werden in der Listenansicht angezeigt.

Seiten erstellen

Um neue Seiten zu erstellen, müssen Sie einen Seitentyp wählen, der die Grundstruktur und die Konfigurationsparameter definiert. Sie können dazu entweder den Namen einer Seite in der Baumansicht anklicken, um die Seite als Stammseite für eine neue Seite auszuwählen, oder Sie werden aufgefordert, eine Stammseite direkt im Dialog Neue Seite zu wählen.

Um eine neue Seite zu erzeugen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten.

    Die Seiten-Baumansicht wird angezeigt.

  2. Klicken Sie Neu.

    Der Dialog Neue Seite wird angezeigt.

  3. Geben Sie einen Namen für die neue Seite an.

    Falls erforderlich, modifizieren Sie die automatisch erzeugte ID.

  4. Wählen Sie eine Stammseite aus.

    Mit einem Klick auf das Feld zur Dateiauswahl wird die Baumansicht für Seiten geöffnet, um die gewünschte Seite auszuwählen.

  5. Wählen Sie einen Seitentyp aus.

    Hinweis:

    Die Seitentypen von system-verwalteten Seiten sind für benutzer-generierte Seiten nicht verfügbar.

  6. Klicken Sie Übernehmen.

    Die neue Seite wird angelegt und in der Detailansicht zur Bearbeitung geöffnet, siehe Seiten bearbeiten.

Seiten sperren

Bevor Sie eine Seite bearbeiten, sollten Sie sie sperren. Die Sperre verhindert, dass die Seite zur gleichen Zeit von anderen Benutzern bearbeitet werden kann. Die Seitensperre kann manuell gelöst werden. Nach Ablauf eines festgelegten Intervalls wird die Produktsperre automatisch aufgehoben. Information zum Konfigurieren der Seitensperre, siehe Zeitraum für Content-Sperre festlegen.

Tip:

Beim Bearbeiten einer Seite wird die Seite automatisch für Sie gesperrt, falls sie nicht schon vorher von einem anderen Benutzer gesperrt wurde.

Seiten können auf verschiedene Weise gesperrt werden: entweder direkt über der Seitenliste/Baumansicht oder über die Seiten-Detailansicht.

Sperren von Seiten über die Seiten-Listen-/Baumansicht

Um Seiten über die Seiten-Listen-/Baumansicht zu sperren:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten.

    Die Seiten-Baumansicht wird angezeigt.

  2. Klicken Sie auf das Symbol in der Zeile der Seite(n), die Sie bearbeiten wollen.

    Falls die Seite gegenwärtig von einem anderen Benutzer bearbeitet wird, erscheint eine Fehlermeldung.

Hinweis:

Klicken Sie auf das Symbol um die Seitensperre aufzuheben, wenn Sie mit der Bearbeitung fertig sind.

Sperren von Seiten in der Detailansicht

Um eine Seite über die Seiten-Detailansicht zu sperren:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten.

    Die Seiten-Baumansicht wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID der Seite, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Klicken Sie auf das Symbol oben im Dialog.

    Falls die Seite gegenwärtig von einem anderen Benutzer bearbeitet wird, erscheint eine Fehlermeldung.

Hinweis:

Klicken Sie auf das Symbol um die Seitensperre aufzuheben, wenn Sie mit der Bearbeitung fertig sind.

Seiten bearbeiten

Allgemeine Optionen zur Bearbeitung von Seiten sind Seiten-Content bearbeiten, Seiteneinstellungen festlegen und (benutzer-generierte) Seiten kopieren. Vor dem Bearbeiten müssen die Seiten gesperrt werden, siehe Seiten sperren.

Seiten-Content bearbeiten

Um den Inhalt von Seiten zu bearbeiten:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten.

    Die Seiten-Baumansicht wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID der Seite, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

    Tip:

    Das Register Content zeigt eine "Komfort"-Ansicht für den Content der Seitenvariante mit der höchsten Priorität, siehe Priorität von Seitenvarianten festlegen.

  3. Legen Sie die erforderlichen Parameter-Werte fest und bearbeiten Sie den Content wie erforderlich.

    Die verfügbaren Parameter oder Content-Elemente hängen vom Template der entsprechenden Seitenvariante ab.

  4. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

    Anderenfalls klicken Sie auf Zurücksetzen, um die Änderungen zu verwerfen.

Seiten kopieren

Wenn Seiten kopiert werden, so werden auch alle zugehörigen Seitenvarianten kopiert.

Hinweis:

System-verwaltete Seiten können nicht kopiert werden.

Um eine Seite zu kopieren:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten.

    Die Seiten-Baumansicht wird angezeigt.

  2. Klicken Sie auf das Symbol in der Zeile der Seite(n), die Sie kopieren wollen.

    Der Dialog Seite kopieren wird angezeigt.

  3. Bearbeiten Sie Namen, ID und Beschreibung wie erforderlich.

  4. Klicken Sie Übernehmen.

    Die Kopie der Seite wird angelegt und in der Detailansicht geöffnet.

Seiteneigenschaften festlegen

Um die Eigenschaften einer Seite festzulegen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten.

    Die Seiten-Baumansicht wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID der Seite, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Wechseln Sie zum Register Eigenschaften.

    Das Register Eigenschaften ermöglicht die Bearbeitung des Anzeigennamens und der Beschreibung einer gewählten Locale sowie Bearbeitung der Stammseite.

  4. Bearbeiten Sie die Eigenschaften wie erforderlich.

  5. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

    Anderenfalls klicken Sie auf Zurücksetzen, um die Änderungen zu verwerfen.

Seitenhierarchie bearbeiten

Die Baumstruktur-Ansicht repräsentiert die Seitenstruktur der aktuellen Seite. Content-Redakteure können die Baumstruktur durch das explizite Festlegen von Stammseiten bearbeiten.

Um die Stammseite einer Seite festzulegen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten.

    Die Seiten-Baumansicht wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID der Seite, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Wechseln Sie zum Register Eigenschaften.

  4. Im Feld Stamm können Sie den gewünschten Stammseiten-Namen festlegen oder mit dem Feld Datei auswählen die gewünschte Stammseite wählen.

    Mit einem Klick auf das Feld zur Dateiauswahl wird ein Dialog zur Auswahl der Stammseite geöffnet.

  5. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

    Damit wird die aktuelle Seite unter die gewählte Stammseite in der Baumansicht geschoben.

    Anderenfalls klicken Sie auf Zurücksetzen, um die Änderungen zu verwerfen.

Seitenvarianten zu Seite hinzufügen

Das Register Seitenvarianten der Detailansicht ermöglicht das explizite Zuordnen einer Seitenvariante zu einer Seite.

Hinweis:

Nur publizierte Seitenvarianten können online sichtbar sein, unabhängig von ihren Seiten-Zuordnungen.

Verfügbare Seitenvarianten beschränken

Um die Seitenvarianten für mögliche Zuordnungen zu beschränken:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten.

    Die Seiten-Baumansicht wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID der Seite, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Öffnen Sie das Register Beschränkungen.

    Die Liste mit Seiten-Templates wird gezeigt, auf deren Basis sie der aktuellen Seite Seitenvarianten zuordnen können.

  4. Wählen Sie die Seiten-Templates, die Sie zulassen wollen.

    Markieren Sie das/die Kontrollkästchen bzw. entfernen Sie die Markierung der entsprechenden Seiten-Templates.

  5. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

    Anderenfalls klicken Sie auf Zurücksetzen, um die Änderungen zu verwerfen.

Hinweis:

Ein ausgewähltes Kontrollkästchen macht die Seitenvarianten aufgrund des entsprechenden Seiten-Templates für die Zuordnung zur aktuellen Seite verfügbar. Ein unmarkiertes Kontrollkästchen beschränkt jedoch die Zuordnung von Seitenvarianten zur aktuellen Seite aufgrund des zugehörigen Seiten-Templates.

Bestehende Seitenvarianten hinzufügen

Um eine bestehende Seitenvariante einer Seite zuzuordnen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten.

    Die Seiten-Baumansicht wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID der Seite, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Wechseln Sie zum Register Seitenvarianten.

    Diese Liste enthält alle dieser Seite zugeordneten Seitenvarianten in einer Listenansicht.

  4. Klicken Sie auf Zuordnen.

    Der Dialog Seitenvariante zuordnen wird angezeigt. Unterhalb des Abschnitts Sichtbarkeit listet der Dialog die verfügbaren Seitenvarianten auf. Falls erforderlich, können Sie die Liste durch Angabe eines Seitennamens oder -typs eingrenzen.

  5. Legen Sie den Sichtbarkeitszeitraum und die Priorität fest.

  6. Wählen Sie die Seitenvarianten, die der Seite zugeordnet werden sollen.

  7. Klicken Sie auf Zuordnen.

    Die Seitenvarianten werden der aktuellen Seite zugeordnet.

Neue Seitenvarianten hinzufügen

Um eine neue Seitenvariante zu erstellen, die direkt mit der aktuellen Seite verbunden ist:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten.

    Die Seiten-Baumansicht wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID der Seite, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Wechseln Sie zum Register Seitenvarianten.

    Diese Liste enthält alle dieser Seite zugeordneten Seitenvarianten in einer Listenansicht.

  4. Klicken Sie Neu.

    Der Dialog Neue Seitenvariante zuordnen wird angezeigt.

  5. Bearbeiten Sie die Details der Seitenvariante wie erforderlich.

    Geben Sie einen Namen an, bearbeiten Sie die ID, falls erforderlich, wählen Sie ein Seiten-Template aus, definieren Sie den Publikationszeitraum und legen Sie den Sichtbarkeitszeitraum und die Priorität fest.

  6. Klicken Sie Übernehmen.

    Die neue Seitenvariante wird angelegt und in der Detailansicht zur Bearbeitung geöffnet, siehe Seitenvarianten bearbeiten.

    Um zur Seiten-Detailansicht zurückzukehren, klicken Sie auf Zurück zur Liste.

Seitenvarianten verschieben

Zugeordnete Seitenvarianten können von einer Seite zu einer anderen Seite verschoben werden. Um dies zu tun:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten.

    Die Seiten-Baumansicht wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID der Seite, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Wechseln Sie zum Register Seitenvarianten.

    Diese Liste enthält alle dieser Seite zugeordneten Seitenvarianten in einer Listenansicht.

  4. Markieren Sie die Zuordnung, die Sie verschieben wollen.

    Markieren Sie das Kontrollkästchen der gewünschten Seitenvarianten-Zuordnung.

  5. Klicken Sie auf Auswahl bearbeiten.

    Die verfügbaren Stapelprozesse werden angezeigt.

  6. Wählen Sie Ausgewählte Seitenvariante Seiten zuordnen oder Ausgewählte Seitenvarianten zu Seiten verschieben, klicken Sie dann Weiter.

    Der Assistent zum Zuordnen der aktuellen Seitenvariante zu einer bestehenden Seite wird geöffnet.

    • Mit "Zuordnen" ordnen Sie die Seitenvariante der neuen Seite zu und die ursprüngliche Zuordnung wird beibehalten.

    • Mit "Verschieben" entfernen Sie die Zuordnung der Seitenvariante zur ursprünglichen Seite und die Seitenvariante wird nur zu der neuen Seite zugeordnet.

  7. Markieren Sie die gewünschte Seite und klicken Sie auf Weiter.

  8. Legen Sie den Sichtbarkeitszeitraum und die Priorität fest und klicken Sie auf Fertigstellen.

    Die Seitenvariante wird der neuen Zielseite zugeordnet und die Zuordnung zur vorherigen Seite wird aufgehoben.

Priorität von Seitenvarianten festlegen

Falls einer Seite mehrere Seitenvarianten mit überlappenden Sichtbarkeitszeiträumen zugeordnet sind, bestimmt die Priorität, welche der online gesetzten Seitenvarianten tatsächlich angezeigt wird. Um die Priorität von zugeordneten Seitenvarianten festzulegen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten.

    Die Seiten-Baumansicht wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID der Seite, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Wechseln Sie zum Register Seitenvarianten.

    Diese Liste enthält alle dieser Seite zugeordneten Seitenvarianten in einer Listenansicht.

  4. Verwenden Sie in der Liste der Seitenvarianten die Pfeiltasten in der Spalte Priorität, um die Prioritäen der Seitenvariante zu ändern.

    Durch Klick auf einen Pfeil wird die entsprechende Seitenvariante nach oben oder unten verschoben, wodurch unmittelbar die Anzeigepriorität geändert wird.

Sichtbarkeit von Seitenvarianten festlegen

Die Sichtbarkeit bezieht sich auf die Verfügbarkeit der Seitenvariante in einem bestimmten Kontext, d.h. innerhalb einer Seite. Die Sichtbarkeit legt die Gültigkeitsdauer einer Seitenvariante-Seite-Zuordnung fest. Falls einer Seite mehrere Seitenvarianten mit überlappenden Sichtbarkeitszeiträumen zugeordnet sind, bestimmt die Priorität, welche der online gesetzten Seitenvarianten tatsächlich angezeigt wird. Damit Seitenvarianten in der Storefront sichtbar werden, müssen die Publikationsdauer sowie der Sichtbarkeitszeitraum und die Priorität übereinstimmen.

Um die Sichtbarkeit von zugeordneten Komponenten festzulegen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten.

    Die Seiten-Baumansicht wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID der Seite, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Wechseln Sie zum Register Seitenvarianten.

    Diese Liste enthält alle dieser Seite zugeordneten Seitenvarianten in einer Listenansicht.

  4. Klicken Sie auf den Sichtbarkeitszeitraum der entsprechenden Seitenvariante.

    Das Dialogfeld Priorität und Sichtbarkeit wird geöffnet.

  5. Bearbeiten Sie den Sichtbarkeitszeitraum und die Priorität wie gewünscht.

  6. Klicken Sie auf OK, um Änderungen zu speichern.

    Anderenfalls klicken Sie auf Abbrechen, um die Änderungen zu verwerfen.

Zuordnungen von Seitenvarianten zu Seiten aufheben

Um die Zuordnung einer Seitenvariante zu einer Seite aufzuheben:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten.

    Die Seiten-Baumansicht wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID der Seite, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Wechseln Sie zum Register Seitenvarianten.

    Diese Liste enthält alle dieser Seite zugeordneten Seitenvarianten in einer Listenansicht.

  4. Markieren Sie Zuordnung, die Sie aufheben wollen.

    Markieren Sie das Kontrollkästchen der gewünschten Seitenvarianten-Zuordnung.

  5. Klicken Sie auf Aufheben und bestätigen Sie die Operation.

    Die Zuordnung des der Seitenvariante zur aktuellen Seite wird entfernt.

Seiten-Label verwalten

Label ermöglichen die Definition von Gruppen abhängiger Objekte, um dann bestimmte Aktionen mit diesen Objekten auszuführen. Informationen zum Erzeugen von Labeln, siehe Label anlegen. Für weitere Informationen zur Ausführung label-basierter Aktionen, siehe Aktionen für Objekte mit Labeln durchführen.

Label zu Seiten hinzufügen

Um einer Seite ein Label zuzuordnen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten.

    Die Seiten-Baumansicht wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID der Seite, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Öffnen Sie das Register Label.

    Eine Liste der Label, die dieser Seite zugeordnet sind, wird angezeigt.

  4. Klicken Sie auf Neu, um eine neue Label-Zuordnung zu definieren.

    Eine Liste der verfügbaren Label wird angezeigt.

  5. Markieren Sie das gewünschte Label und klicken Sie auf OK.

    Das ausgewählte Label wird der aktuellen Seite zugeordnet.

Label von Seiten entfernen

Um die Zuordnung eines Labels zu einer Seite aufzuheben:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten.

    Die Seiten-Baumansicht wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID der Seite, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Öffnen Sie das Register Label.

    Eine Liste der Label, die dieser Seite zugeordnet sind, wird angezeigt.

  4. Markieren Sie das Kontrollkästchen der Labelzuordnung, die Sie aufheben wollen.

  5. Klicken Sie auf Aufheben und bestätigen Sie die Operation.

    Die Zuordnung des Labels zur aktuellen Seite wird entfernt.

Seiten in Stapelprozessen bearbeiten

Bestimmte Operationen in der Seitenverwaltung können mittels Stapelprozessen mit mehreren Seiten gleichzeitig ausgeführt werden. Operationen, die als Stapelprozesse ausgeführt werden können, sind:

  • Seitenvarianten ausgewählten Seiten zuordnen

  • Ausgewählten Seiten Stamm zuordnen

  • Ausgewählten Seiten Label zuordnen

  • Markierte Seiten löschen

Weitere Informationen zum Bearbeiten von Seiten per Stapelprozess, siehe Objekte in Stapelprozessen bearbeiten.

Publikationsseiten

In einer Umgebung der Datenreplizierung müssen Seiten, die im Bearbeitungssystem erzeugt werden, im Zielsystem veröffentlicht werden. Dies geschieht entweder automatisch im Kontext von regulärer Datenreplizierung (siehe Verwaltung von Replikationsaufgaben) oder manuell, wie unten beschrieben.

Um Seiten manuell zu publizieren, gehen Sie wie folgt vor:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten.

    Die Seiten-Baumansicht wird angezeigt.

  2. Markieren Sie die Kontrollkästchen der zu publizierenden Seiten in der Listenansicht.

    Alternativ können Sie eine Suche ausführen und auf Alles auswählen klicken, um das Suchergebnis auszuwählen.

  3. Klicken Sie Publizieren im Zielsystem.

    Die Liste konfigurierter Zielsysteme wird angezeigt.

  4. Wählen Sie das gewünschte Zielsystem und die Lösung für einen Sperrkonflikt.

    Für Informationen zu den Sperrkonflikt-Lösungen, siehe Objektsperren.

  5. Klicken Sie auf Fertigstellen.

    Die gewählten Seiten werden an das gewählte Zielsystem repliziert.
