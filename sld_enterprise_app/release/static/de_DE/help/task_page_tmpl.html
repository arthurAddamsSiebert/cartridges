Content-Redakteure können im Commerce Management modell-basierte Seiten-Templates anlegen und anpassen. Für Informationen zu Seiten-Templates und anderen Content-Management-Konzepten siehe Content-Management: Begriffe.

Hinweis:

Das Erzeugen von Template-Modellen erfordert Entwicklungsaufwände.

Seiten-Templates können von der Organisation freigegeben werden und auf Channel-Ebene oder Applikationsebene erstellt, bearbeitet etc. werden.

Seiten-Templates Anzeigen

Um alle verfügbaren Seiten-Templates für einen bestimmten Kontext anzuzeigen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox.

    Die Navigationsleiste für den Managementkontext wird geöffnet.

  2. Klicken Sie in der Navigationsleiste auf Content | Seiten-Templates.

    Die Baumansicht für Seiten-Templates wird angezeigt.

  3. Wählen Sie den Applikationstyp aus (optional).

    Im Channel- oder Organisationskontext zeigt die Liste alle Seiten-Templates aller im jeweiligen Kontext verfügbaren Applikationstypen an. Mit der Auswahl eines Applikationstyps legen Sie das Content-Modell, mit dem Sie arbeiten werden, fest (siehe Was ist eine Applikation?) und filtern die Seiten-Templates entsprechend.

    Abbildung 43: Auswahl des Content-Modells
    Tip:

    Diese Einstellung ist während Ihrer gesamten Commerce Management-Sitzung aktiv. Um die Auswahl aufzuheben, wählen Sie einen anderen Applikationstyp oder "Alle" aus.

Für eine Einführung in das Modul Seiten-Templates, siehe Modul Seiten-Template.

Seiten-Templates suchen

In der einfachen Suche lassen sich Seiten-Templates nach Name oder ID suchen. Falls erforderlich, können Sie die Suche nach Seiten-Templates auf einen bestimmten Seitenvarianten-Typ oder eine Gruppe eingrenzen. Die Suche nach Seiten-Templates ist über die Listenansicht der Seiten-Templates verfügbar.

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Templates.

    Die Baumansicht für Seiten-Templates wird angezeigt.

  2. Öffnen Sie das Register Listenansicht.

  3. Geben Sie den Namen des Seiten-Templates oder die ID als Suchbegriff ein.

    Falls erforderlich, wählen Sie einen Seitenvarianten-Typ aus, um die Suche einzugrenzen.

  4. Klicken Sie auf Suchen.

    Die Suchergebnisse werden in der Listenansicht angezeigt.

Seiten-Templates erstellen

Um neue Seiten-Templates zu erstellen, müssen Sie ein Stamm-Template wählen, das die Grundstruktur und die Konfigurationsparameter definiert. Sie werden direkt im Dialog Neues Seiten-Template aufgefordert, einen Stamm auszuwählen.

Hinweis:

Die Templates der "obersten Ebene", die nach Systemstart verfügbar sind, stellen verfügbare Seitenvarianten-Typen dar.

Um ein neues Seiten-Template zu erstellen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Templates.

    Die Baumansicht für Seiten-Templates wird angezeigt.

  2. Klicken Sie Neu.

    Der Dialog Neues Seiten-Template wird angezeigt.

  3. Geben Sie einen Namen für das neue Seiten-Template an.

    Falls erforderlich, modifizieren Sie die automatisch erzeugte ID.

  4. Wählen Sie ein Stamm-Seiten-Template.

    Mit einem Klick auf das Feld zur Dateiauswahl wird die Baumansicht für Seiten-Templates geöffnet, um das gewünschte Seiten-Template auszuwählen.

  5. Klicken Sie Übernehmen.

    Das neue Seiten-Template wird angelegt und in der Detailansicht zur Bearbeitung geöffnet, siehe Seiten-Templates bearbeiten.

Seiten-Templates sperren

Bevor Sie ein Seiten-Template bearbeiten, sollten Sie es sperren. Die Sperre verhindert, dass das Seiten-Template zur gleichen Zeit von anderen Benutzern bearbeitet werden kann. Die Sperre kann manuell gelöst werden. Nach Ablauf eines festgelegten Intervalls wird die Produktsperre automatisch aufgehoben. Information zum Konfigurieren der Sperre von Seiten-Templates finden Sie in Zeitraum für Content-Sperre festlegen.

Tip:

Beim Bearbeiten eines Seiten-Templates wird dieses automatisch für Sie gesperrt, falls es nicht schon vorher von einem anderen Benutzer gesperrt wurde.

Seiten-Templates können auf verschiedene Weise gesperrt werden: entweder direkt über der Listen-/Baumansicht für Seiten-Templates oder über die Seiten-Template-Detailansicht.

Sperren von Seiten über die Seiten-Listen-/Baumansicht

Um Seiten über die Listen-/Baumansicht zu sperren:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Templates.

    Die Baumansicht für Seiten-Templates wird angezeigt.

  2. Klicken Sie auf das Symbol in der Zeile der Seiten-Template(s), die Sie bearbeiten wollen.

    Falls das Seiten-Template gegenwärtig von einem anderen Benutzer bearbeitet wird, erscheint eine Fehlermeldung.

Hinweis:

Klicken Sie auf das Symbol , um die Sperre des Seiten-Templates aufzuheben, wenn Sie mit der Bearbeitung fertig sind.

Sperren von Seiten-Templates in der Detail-Ansicht

Um eine Seite über die Template-Detailansicht zu sperren:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Templates.

    Die Baumansicht für Seiten-Templates wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID des Seiten-Templates, das Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Klicken Sie auf das Symbol oben im Dialog.

    Falls das Seiten-Template gegenwärtig von einem anderen Benutzer bearbeitet wird, erscheint eine Fehlermeldung.

Hinweis:

Klicken Sie auf das Symbol , um die Sperre des Seiten-Templates aufzuheben, wenn Sie mit der Bearbeitung fertig sind.

Seiten-Templates bearbeiten

Allgemeine Optionen zur Bearbeitung von Seiten-Templates sind Bearbeitung von Seiten-Template-Content, Seiten-Template-Einstellungen festlegen und (benutzer-generierte) Seiten-Templates kopieren. Vor dem Bearbeiten müssen die Seiten-Templates gesperrt werden, siehe Seiten-Templates sperren.

Content von Seiten-Templates bearbeiten

Um den Inhalt von Seiten-Templates zu bearbeiten:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Templates.

    Die Baumansicht für Seiten-Templates wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID des Seiten-Templates, das Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet. Die Übersichtsseite des Registers Content zeigt einen Strukturbaum, der den Ursprung des aktuellen Elements und eine Platzhalter-Liste als Referenz aller zugeordneten Platzhaltern.

  3. Legen Sie die erforderlichen Parameter-Werte fest und bearbeiten Sie den Content wie erforderlich.

    Die verfügbaren Parameter oder Content-Elemente hängen vom Seitenvarianten-Typ ab, auf dem das aktuelle Template basiert.

    Tip:

    Parameterwerte, die aus Stammobjekten vererbt wurden, können bearbeitet werden, siehe Ererbte Parameter bearbeiten.

  4. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

    Anderenfalls klicken Sie auf Zurücksetzen, um die Änderungen zu verwerfen.

Ererbte Parameter bearbeiten

Es werden standardmäßig alle Parameterwerte aus den Stammobjekten vererbt. Jedoch können sie auf jeder Vererbungs-Ebene überladen sein. Überladene Werte können auf den ursprünglichen (ererbten) Wert entsprechend zurückgesetzt werden.

Ererbte Parameter Überladen

Um einen ererbten Parameterwert zu überladen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Templates.

    Die Baumansicht für Seiten-Templates wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID des Seiten-Templates, das Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Beginnen Sie im Abschnitt Parameter mit der Bearbeitung der gewünschten Parameterwerte.

    Wird ein Wert bearbeitet, so zeigt ein Stift-Symbol () neben dem Wert an, dass dieser Wert nicht mehr ererbt, sondern überladen ist.

  4. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

    Anderenfalls klicken Sie auf Zurücksetzen, um die Änderungen zu verwerfen.

Überladen von Parametern Zurücksetzen

Um einen überladenen Parameter auf seine ursprünglichen (ererbten) Wert zurückzusetzen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Templates.

    Die Baumansicht für Seiten-Templates wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID des Seiten-Templates, das Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Klicken Sie im Abschnitt Parameter auf Zurücksetzen auf geerbte Werte.

    Es werden Kontrollkästchen vor den bearbeiteten Parametern angezeigt.

  4. Wählen Sie die Parameter, die zurückgesetzt werden sollen.

    Verwenden Sie die Kontrollkästchen, um die gewünschten Parameter auszuwählen.

  5. Klicken Sie auf Zurücksetzen.

    Die ausgewählten Parameter werden auf ihre ursprünglichen (ererbten) Werte zurückgesetzt.

Seiten-Templates kopieren

Wenn Seiten-Templates kopiert werden, werden auch alle zugehörigen Komponenten und Platzhalter kopiert.

Hinweis:

System-verwaltete Seiten-Templates können nicht kopiert werden.

Um ein Seiten-Template zu kopieren:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Templates.

    Die Baumansicht für Seiten-Templates wird angezeigt.

  2. Klicken Sie auf das Symbol in der Zeile des Seiten-Templates, das Sie kopieren wollen.

    Der Dialog Seite-Template kopieren wird angezeigt.

  3. Bearbeiten Sie Namen, ID und Beschreibung wie erforderlich.

  4. Klicken Sie Übernehmen.

    Die Kopie des Seiten-Templates wird angelegt und in der Detailansicht geöffnet.

Eigenschaften von Seiten-Templates festlegen

Um die Eigenschaften eines Seiten-Templates festzulegen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Templates.

    Die Baumansicht für Seiten-Templates wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID des Seiten-Templates, das Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Wechseln Sie zum Register Eigenschaften.

    Das Register Eigenschaften ermöglicht die Bearbeitung des Anzeigennamens und der Beschreibung einer gewählten Locale.

  4. Bearbeiten Sie die Eigenschaften wie erforderlich.

  5. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

    Anderenfalls klicken Sie auf Zurücksetzen, um die Änderungen zu verwerfen.

Komponenten/Komponenten-Templates zu Seiten-Templates hinzufügen

Das Register Content der Detailansicht ermöglicht das explizite Zuordnen einer Komponente oder eines Komponenten-Templates zu einem Seiten-Template.

Hinweis:

Nur publizierte Komponenten können online sichtbar sein, unabhängig von ihren Seitenbereich-Zuordnungen.

Bestehende Komponenten/Komponenten-Templates hinzufügen

Um eine bestehende Komponente oder ein Komponenten-Template in ein Seiten-Template einzufügen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Templates.

    Die Baumansicht für Seiten-Templates wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID des Seiten-Templates, das Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Klicken Sie im Abschnitt Content auf Zuordnen.

    Der Dialog Komponente/Komponenten-Template zuordnen wird angezeigt. Unterhalb des Abschnitts Sichtbarkeit listet der Dialog die verfügbaren Komponenten und Komponenten-Templates auf.

  4. Legen Sie den Sichtbarkeitszeitraum und die Priorität fest.

  5. Wählen Sie die Komponenten oder Komponenten-Templates aus, die dem Seiten-Template zugeordnet werden sollen.

  6. Klicken Sie auf Zuordnen.

    Die Komponenten oder Komponenten-Templates werden dem aktuellen Seiten-Template zugeordnet.

Neue Komponenten/Komponenten-Templates hinzufügen

Um eine neue Komponente oder ein Komponenten-Template zu erzeugen, das dem aktuellen Seiten-Template direkt zugeordnet ist:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Templates.

    Die Baumansicht für Seiten-Templates wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID des Seiten-Templates, das Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Klicken Sie im Abschnitt Content auf Neu.

    Der Dialog Neue Komponente/Neues Komponenten-Template wird angezeigt.

  4. Bearbeiten Sie die Details der Komponente/des Komponenten-Templates wie erforderlich.

    Geben Sie einen Namen an, bearbeiten Sie die ID, falls erforderlich, wählen Sie ein Komponenten-Template aus und legen Sie den Sichtbarkeitszeitraum und die Priorität fest. Markieren Sie das Feld Aktiviert für ein neues Komponenten-Template wie erforderlich. Definieren Sie für eine neue Komponente den Publikationszeitraum.

    Hinweis:

    Wählen Sie, ob das neue Element ein Komponenten-Template oder eine Komponente ist.

  5. Klicken Sie Übernehmen.

    Die neue Komponente bzw. das Komponenten-Template wird angelegt und in der entsprechenden Detailansicht zur Bearbeitung geöffnet, siehe Komponenten bearbeiten oder Seiten-Templates bearbeiten.

    Um zur Detailansicht des Seiten-Templates zurückzukehren, klicken Sie auf Zurück zur Liste.

Prioritäten von Komponten(-Templates) festlegen

Falls einem Seiten-Template mehrere Komponenten(-Templates) mit überlappenden Sichtbarkeitszeiträumen zugeordnet sind, bestimmt die Priorität, in welcher Reihenfolge die online gesetzten Komponenten(-Templates) angezeigt werden sollen. Um die Priorität von zugeordneten Komponenten(-Templates) festzulegen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Templates.

    Die Baumansicht für Seiten-Templates wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID des Seiten-Templates, das Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Benutzen Sie im Abschnitt Content in der Liste der Komponenten(-Templates) die Pfeile in der Spalte Priorität, um die Priorität der Komponenten(-Templates) zu ändern.

    Durch Klick auf eine Pfeil-Schaltfläche wird die entsprechende Komponente bzw. das Template nach oben oder unten verschoben, wodurch unmittelbar die Anzeigepriorität geändert wird.

Sichtbarkeit für Komponenten(-Templates) festlegen

Die Sichtbarkeit bezieht sich auf die Verfügbarkeit der Komponente/des Komponenten-Templates in einem bestimmten Kontext, d.h. innerhalb eines Seiten-Templates. Die Sichtbarkeit legt die Gültigkeitsdauer einer Zuordnung zwischen Komponenten(-Template) und Seiten-Template fest. Falls einem Seiten-Template mehrere Komponenten(-Templates) mit überlappenden Sichtbarkeitszeiträumen zugeordnet sind, bestimmt die Priorität, in welcher Reihenfolge die online gesetzten Komponenten(-Templates) angezeigt werden sollen. Damit Komponenten(-Templates) in der Storefront sichtbar werden, müssen die Publikationsdauer sowie der Sichtbarkeitszeitraum und die Priorität übereinstimmen.

Um die Sichtbarkeit von zugeordneten Komponenten(-Templates) festzulegen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Templates.

    Die Baumansicht für Seiten-Templates wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID des Seiten-Templates, das Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Klicken Sie im Abschnitt Content auf das Sichtbarkeitsdatum der entsprechenden Komponente/des Komponenten-Templates.

    Das Dialogfeld Priorität und Sichtbarkeit wird geöffnet.

  4. Bearbeiten Sie den Sichtbarkeitszeitraum und die Priorität für Komponente/Komponenten-Template wie gewünscht.

  5. Klicken Sie auf OK, um Änderungen zu speichern.

    Anderenfalls klicken Sie auf Abbrechen, um die Änderungen zu verwerfen.

Zuordnung von Komponenten(-Templates) zu Seiten-Templates aufheben

Um die Zuordnung einer Komponente/eines Komponenten-Templates zu einem Seiten-Template aufzuheben:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Templates.

    Die Baumansicht für Seiten-Templates wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID des Seiten-Templates, das Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Wählen Sie im Register Content die Zuordnung aus, die Sie entfernen wollen.

    Markieren Sie das Kontrollkästchen der gewünschten Komponenten(-Template)-Zuordnung.

  4. Klicken Sie auf Aufheben und bestätigen Sie die Operation.

    Die Zuordnung der Komponente/des Komponenten-Templates zum gewählten Seiten-Template wird entfernt.

Platzhalter verwalten

Das Register Content der Detailansicht ermöglicht das Verwalten von Platzhaltern eines Seiten-Templates.

Platzhalter hinzufügen

Um einem Seiten-Template einen Platzhalter hinzuzufügen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Templates.

    Die Baumansicht für Seiten-Templates wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID des Seiten-Templates, das Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Klicken Sie im Abschnitt Platzhalter auf das Symbol Symbol.

    Der Dialog Platzhalter wird angezeigt.

    Tip:

    Mithilfe des Links Platzhalter erstellen in dem gewünschten Zielabschnitt können Sie einen Platzhalter erstellen. Somit werden der Typ und die Platzhalter-Zuordnung bereits festgelegt.

  4. Bearbeiten Sie die Details wie erforderlich.

    Geben Sie einen Namen an, bearbeiten Sie die ID wie erforderlich und wählen Sie einen Seitenbereich-Typ.

    Tip:

    In einem Seiten-Template, welches von einem benutzer-definierten Seiten-Template stammt, können Sie den aktuellen Platzhalter einem Platzhalter desselben Typs aus dem Stamm-Template zuordnen. Wählen Sie dazu im Feld Platzhalterzuordnung den gewünschten "Stamm-"Platzhalter aus.

  5. Klicken Sie Übernehmen.

    Der Platzhalter wird erzeugt und eine Liste mit Komponenten, die für die Zuordnung zulässig sind, wird angezeigt.

    Standardmäßig sind alle zulässigen Komponenten ausgewählt. Informationen zum Beschränken der Auswahl erlaubter Komponenten, siehe Platzhalter-Beschränkungen definieren.

Platzhalter-Details bearbeiten

Um Details eines Platzhalters zu bearbeiten:

  1. Gehen Sie zum betreffenden Platzhalter.

    Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Templates. Klicken Sie auf den Namen oder die ID des Seiten-Templates, das Sie bearbeiten wollen, um die Detailansicht zu öffnen, und scrollen Sie zum Abschnitt Platzhalter.

  2. Klicken Sie auf das Symbol neben dem Platzhalter, den Sie bearbeiten wollen.

    Die Platzhalter-Detailansicht wird angezeigt.

  3. Bearbeiten Sie die Details wie erforderlich und klicken Sie auf Übernehmen.

    Klicken Sie auf Zurück, um zurück zur Template-Detailansicht zu gelangen.

Platzhalter-Beschränkungen definieren

Um die Auswahl der in einem Platzhalter erlaubten Komponenten einzuschränken:

  1. Gehen Sie zum betreffenden Platzhalter.

    Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Templates. Klicken Sie auf den Namen oder die ID des Seiten-Templates, das Sie bearbeiten wollen, um die Detailansicht zu öffnen, und scrollen Sie zum Abschnitt Platzhalter.

  2. Klicken Sie auf das Symbol neben dem Platzhalter, den Sie bearbeiten wollen.

    Die Platzhalter-Detailansicht wird angezeigt.

  3. Öffnen Sie das Register Beschränkungen.

    Die verfügbaren Elemente werden angezeigt.

  4. Um die für den aktuellen Platzhalter zulässigen Komponenten festzulegen, markieren Sie die entsprechenden Kontrollkästchen.

    Ein ausgewähltes Kontrollkästchen macht die Komponenten aufgrund des entsprechenden Komponenten-Templates für die Zuordnung zum aktuellen Platzhalter verfügbar. Ein unmarkiertes Kontrollkästchen beschränkt jedoch die Zuordnung von Komponenten zum aktuellen Platzhalter aufgrund des zugehörigen Komponenten-Templates.

  5. Klicken Sie Übernehmen.

    Klicken Sie auf Zurück, um zurück zur Template-Detailansicht zu gelangen.

Platzhalter löschen

Um einen Platzhalter von einem Seiten-Template zu löschen:

  1. Gehen Sie zum betreffenden Platzhalter.

    Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Templates. Klicken Sie auf den Namen oder die ID des Seiten-Templates, das Sie bearbeiten wollen, um die Detailansicht zu öffnen, und scrollen Sie zum Abschnitt Platzhalter.

  2. Klicken Sie auf das Symbol neben dem Platzhalter, den Sie löschen wollen.

    Ein Dialog zur Bestätigung wird angezeigt.

  3. Klicken Sie auf OK, um die Operation zu bestätigen.

    Der gewählte Platzhalter wird vom aktuellen Seiten-Template entfernt.

Hinweis:

Das Löschen eines Platzhalters löscht auch den dem Platzhalter zugewiesenen Content aus den Seitenvarianten und Seiten-Templates, die auf dem aktuellen Template basieren.

Platzhalter anzeigen

Wenn Sie freigegebene Seiten-Templates benutzen, können Sie ausgehende Platzhalter nicht bearbeiten. Um die Platzhalter eines freigegebenen Templates anzuzeigen:

  1. Gehen Sie zum betreffenden Platzhalter.

    Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Templates. Klicken Sie auf den Namen oder die ID des Seiten-Templates, das Sie bearbeiten wollen, um die Detailansicht zu öffnen, und scrollen Sie zum Abschnitt Platzhalter.

  2. Klicken Sie auf das Symbol neben dem Platzhalter, den Sie anzeigen wollen.

    Die Platzhalter-Detailansicht wird im Nur-Lese-Modus angezeigt.

Ausgehende Platzhalter sortieren

Sie können die Sortierung von Platzhaltern definieren, die von einem Seiten-Template nach außen gegeben werden ("ausgehende Platzhalter"). Diese Sortierung legt fest, in welcher Reihenfolge die Platzhalter in Seitenvarianten angezeigt werden, die von diesem Template abgeleitet sind. Um ausgehende Platzhalter zu sortieren:

  1. Gehen Sie zum Abschnitt Ausgehende Platzhalter.

    Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Templates. Klicken Sie auf den Namen oder die ID des Seiten-Templates, das Sie bearbeiten wollen, um die Detailansicht zu öffnen, und scrollen Sie zum Abschnitt Ausgehende Platzhalter.

  2. Klicken Sie auf die Pfeile, um die Platzhalter neu anzuordnen.

    Ihre Änderungen werden sofort wirksam.

Seiten-Template-Label verwalten

Label ermöglichen die Definition von Gruppen abhängiger Objekte, um dann bestimmte Aktionen mit diesen Objekten auszuführen. Informationen zum Erzeugen von Labeln, siehe Label anlegen. Für weitere Informationen zur Ausführung label-basierter Aktionen, siehe Aktionen für Objekte mit Labeln durchführen.

Label zu Seiten-Templates hinzufügen

Um einem Seiten-Template ein Label zuzuordnen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Templates.

    Die Baumansicht für Seiten-Templates wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID des Seiten-Templates, das Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Öffnen Sie das Register Label.

    Eine Liste der Label, die diesem Seiten-Template zugeordnet sind, wird angezeigt.

  4. Klicken Sie auf Neu, um eine neue Label-Zuordnung zu definieren.

    Eine Liste der verfügbaren Label wird angezeigt.

  5. Markieren Sie das gewünschte Label und klicken Sie auf OK.

    Das ausgewählte Label wird dem aktuellen Seiten-Template zugeordnet.

Label von Seiten-Templates entfernen

Um die Zuordnung eines Labels zu einem Seiten-Template aufzuheben:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Templates.

    Die Baumansicht für Seiten-Templates wird angezeigt.

  2. Klicken Sie auf den Namen oder die ID des Seiten-Templates, das Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Öffnen Sie das Register Label.

    Eine Liste der Label, die diesem Seiten-Template zugeordnet sind, wird angezeigt.

  4. Markieren Sie das Kontrollkästchen der Labelzuordnung, die Sie aufheben wollen.

  5. Klicken Sie auf Aufheben und bestätigen Sie die Operation.

    Die Zuordnung des Labels zum aktuellen Seiten-Template wird entfernt.

Seiten-Templates publizieren

In einer Umgebung der Datenreplizierung müssen Seiten-Templates, die im Bearbeitungssystem erzeugt werden, im Zielsystem veröffentlicht werden. Dies geschieht entweder automatisch im Kontext von regulärer Datenreplizierung (siehe Verwaltung von Replikationsaufgaben) oder manuell, wie unten beschrieben.

Um Seiten-Templates manuell zu publizieren, gehen Sie wie folgt vor:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Templates.

    Die Baumansicht für Seiten-Templates wird angezeigt.

  2. Markieren Sie die Kontrollkästchen der zu publizierenden Seiten-Templates in der Listenansicht.

    Alternativ können Sie eine Suche ausführen und auf Alles auswählen klicken, um das Suchergebnis auszuwählen.

  3. Klicken Sie Publizieren im Zielsystem.

    Die Liste konfigurierter Zielsysteme wird angezeigt.

  4. Wählen Sie das gewünschte Zielsystem und die Lösung für einen Sperrkonflikt.

    Für Informationen zu den Sperrkonflikt-Lösungen, siehe Objektsperren.

  5. Klicken Sie auf Fertigstellen.

    Die gewählten Seiten-Templates werden an das gewählte Zielsystem repliziert.
