Content-Redakteure können im Commerce Management modell-basierte Seitenvarianten anlegen, bearbeiten und löschen. Für Informationen zu Seitenvarianten und anderen Content-Management-Konzepten siehe Für Informationen zu Seitenvarianten und anderen Content-Management-Konzepten, siehe Content-Management: Begriffe.

Hinweis:

Das Erzeugen von Seitenvarianten-Modellen erfordert Entwicklungsaufwände.

Seitenvarianten können von der Organisation freigegeben werden und auf Channel-Ebene oder Applikationsebene erstellt, bearbeitet etc. werden.

Seitenvarianten Anzeigen

Um alle verfügbaren Seiten-Varianten für einen bestimmten Kontext anzuzeigen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox.

    Die Navigationsleiste für den Managementkontext wird geöffnet.

  2. Klicken Sie in der Navigationsleiste auf Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  3. Wählen Sie den Applikationstyp aus (optional).

    Im Channel- oder Organisationskontext zeigt die Liste alle Seitenvarianten aller im jeweiligen Kontext verfügbaren Applikationstypen an. Mit der Auswahl eines Applikationstyps legen Sie das Content-Modell, mit dem Sie arbeiten werden, fest (siehe Was ist eine Applikation?) und filtern die Seitenvarianten entsprechend.

    Abbildung 38: Auswahl des Content-Modells
    Tip:

    Diese Einstellung ist während Ihrer gesamten Commerce Management-Sitzung aktiv. Um die Auswahl aufzuheben, wählen Sie einen anderen Applikationstyp oder "Alle" aus.

Für eine Einführung in das Modul Seitenvarianten, siehe Modul Seitenvarianten.

Seitenvarianten suchen

Einfache Seitenvariantensuche

In der einfachen Suche lassen sich Seitenvariante nach Namen oder Teilen des Inhalts suchen. Falls erforderlich, können Sie die Suche auf einen bestimmten Seitentyp oder eine Gruppe eingrenzen.

Um die einfache Suche zu verwenden:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Geben Sie den Namen der Seitenvariante oder einen Teil des Inhalts als Suchbegriff ein.

    Falls erforderlich, wählen Sie einen Seitentyp aus, um die Suche einzugrenzen.

    Hinweis:

    Es ist nicht möglich, nach Content von freigegebenen Seitenvarianten zu suchen, deren Inhalt überschrieben wurde, siehe Steuern von Änderungen an freigegebenem Content. Die Suche liefert nur Treffer aus dem Original-Content.

  3. Klicken Sie auf Suchen.

    Die Suchergebnisse werden in der Listenansicht angezeigt.

Tip:

Standardmäßig wird die einfache Suche angezeigt. Wenn Sie zuvor die erweiterte Suche verwendet haben, klicken Sie auf Einfache Suche.

Erweiterte Seitenvariantensuche

In der Erweiterten Suche können Sie zusätzliche Suchkriterien angeben und so Ihr Suchergebnis verbessern. Um die erweiterte Suchmaske zu verwenden:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Klicken Sie auf Erweiterte Suche.

  3. Geben Sie Ihre Suchkriterien in den entsprechenden Feldern an.

    Die Felder sind selbsterklärend.

    Hinweis:

    Es ist nicht möglich, nach Content von freigegebenen Seitenvarianten zu suchen, deren Inhalt überschrieben wurde, siehe Steuern von Änderungen an freigegebenem Content. Die Suche liefert nur Treffer aus dem Original-Content.

    Tip:

    Produkt- oder Kategoriefilter schränken die Suche auf Seitenvarianten ein, die den damit herausgefilterten Produkten oder Kategorien zugeordnet sind. Das Anlegen von Filtern stellt eine zusätzliche erweiterte Suche nach dem entsprechenden Objekt dar. Sobald der Filter angelegt und angewendet ist, kehren Sie zur ursprünglichen erweiterten Suche nach Seitenvarianten zurück.

  4. Klicken Sie auf Suchen.

    Die Suchergebnisse werden in der Listenansicht angezeigt.

Seitenvarianten erzeugen

Um eine neue Seitenvariante zu erzeugen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Klicken Sie Neu.

    Der Dialog Neue Seitenvariante wird angezeigt.

  3. Geben Sie einen Namen für die neue Seitenvariante an.

    Falls erforderlich, modifizieren Sie die automatisch erzeugte ID.

  4. Wählen Sie das gewünschte Seiten-Template aus.

    Mit einem Klick auf das Feld zur Dateiauswahl wird ein Dialog zur Auswahl des Seiten-Templates geöffnet.

  5. Legen Sie Publikations-Status und -Dauer fest.

  6. Klicken Sie Übernehmen.

    Die neue Seitenvariante wird angelegt und in der Detailansicht zur Bearbeitung geöffnet, siehe Seitenvarianten bearbeiten.

Seitenvarianten sperren

Bevor Sie eine Seitevariante bearbeiten, sollten Sie sie sperren. Die Sperre verhindert, dass die Seitenvariante zur gleichen Zeit von anderen Benutzern bearbeitet werden kann. Die Sperre kann manuell gelöst werden. Nach Ablauf eines festgelegten Intervalls wird die Produktsperre automatisch aufgehoben. Information zum Konfigurieren der Seitenvarianten-Sperre, siehe Zeitraum für Content-Sperre festlegen.

Tip:

Beim Bearbeiten einer Seitenvariante wird diese automatisch für Sie gesperrt, falls sie nicht schon vorher von einem anderen Benutzer gesperrt wurde.

Seitevarianten können auf verschiedene Weise gesperrt werden: entweder direkt über der Liste von Seitenvarianten oder über deren Detailansicht.

Sperren von Seiten-Varianten in der Listenansicht

Um Seitenvarianten über die Listenansicht zu sperren:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Klicken Sie auf das Symbol in der Zeile der Seitenvariante(n), die Sie bearbeiten wollen.

    Falls die Seitenvariante gegenwärtig von einem anderen Benutzer bearbeitet wird, erscheint eine Fehlermeldung.

Hinweis:

Klicken Sie auf das Symbol , um die Sperre aufzuheben, wenn Sie mit der Bearbeitung fertig sind.

Sperren von Seiten-Varianten in der Detailansicht

Um eine Seitenvariante über die Detailansicht zu sperren:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Klicken Sie auf den Namen der Seitenvariante, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Klicken Sie auf das Symbol oben im Dialog.

    Falls die Seitenvariante gegenwärtig von einem anderen Benutzer bearbeitet wird, erscheint eine Fehlermeldung.

Hinweis:

Klicken Sie auf das Symbol , um die Sperre aufzuheben, wenn Sie mit der Bearbeitung fertig sind.

Seitenvarianten bearbeiten

Zu den Bearbeitungsfunktionen für Seitenvarianten gehören das Bearbeiten von Inhalten der Seitenvariante und das Kopieren von Seitenvarianten. Vor dem Bearbeiten müssen die Seitenvarianten gesperrt werden, siehe Seitenvarianten sperren.

Content von Seitenvarianten bearbeiten

Um den Inhalt von Seitenvarianten zu bearbeiten:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Klicken Sie auf den Namen der Seitenvariante, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Legen Sie die erforderlichen Parameter-Werte fest und bearbeiten Sie den Content wie erforderlich.

    Die verfügbaren Parameter oder Content-Elemente hängen vom Seiten-Template ab.

    Tip:

    Parameterwerte, die aus Stammobjekten vererbt wurden, können bearbeitet werden, siehe Ererbte Parameter bearbeiten.

  4. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

    Anderenfalls klicken Sie auf Zurücksetzen, um die Änderungen zu verwerfen.

SEO-Optionen für Seitenvarianten festlegen

Falls erforderlich, können Sie Seitenvarianten die Tags noindex und nofollow hinzufügen, siehe SEO-spezifische Tags. Um dies zu tun:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Klicken Sie auf den Namen der Seitenvariante, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Wählen Sie im Abschnitt Parameter die Optionen No Index und/oder No Follow.

    No Index verhindert, dass Suchmaschinen die zugehörige Seite indexieren.

    No Follow verhindert, dass Suchmaschinen Links auf der zugehörigen Seite verfolgen.

    Hinweis:

    Das Aktivieren dieser Optionen kann die entsprechende Einstellung an der Kategorie oder am Produkt überschreiben.

  4. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

    Anderenfalls klicken Sie auf Zurücksetzen, um die Änderungen zu verwerfen.

Standard-Content von Seitenvarianten verbergen

Seiten-Templates (siehe Verwaltung von Komponenten-Templates) können an abgeleitete Seitenvarianten Platzhalter mit bereits zugewiesenem Standard-Content weitergeben. Beim Bearbeiten der Seitenvarianten können Sie diesen Standard-Content ausblenden. Um dies zu tun:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Klicken Sie auf den Namen der Seitenvariante, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Klicken Sie im Abschnitt Platzhalter auf das Symbol neben dem betreffenden Platzhalter.

    Der Dialog Einstellungen wird angezeigt.

  4. Markieren Sie das Kontrollkästchen Standardcontent ausblenden, und klicken Sie auf Übernehmen.

    Der vom Template abgeleitete Content wird nicht mehr angezeigt.

    Tip:

    Bei Seitenvarianten, die aus einem Stamm-Management-Kontext freigegeben wurden (siehe Freigaben von Content-Elementen), ist die Option, den Standard-Content zu verbergen, nicht verfügbar.

Platzierung von Seitenvarianten-Content steuern

Seiten-Templates (siehe Verwaltung von Komponenten-Templates) können an abgeleitete Seitenvarianten Platzhalter mit bereits zugewiesenem Standard-Content weitergeben. Wenn Sie eigenen Content in diese Platzhalter einfügen, wird dieser normalerweise unter dem Standard-Content positioniert. Sie können dieses Verhalten jedoch ändern und eigenen Content oberhalb des Standard-Contents anzeigen lassen. Um dies zu tun:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Klicken Sie auf den Namen der Seitenvariante, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Klicken Sie im Abschnitt Platzhalter auf das Symbol neben dem betreffenden Platzhalter.

    Der Dialog Einstellungen wird angezeigt.

  4. Markieren Sie das Kontrollkästchen Komponenten über Standard-Content platzieren und klicken Sie auf Übernehmen.

    Lokal zugewiesene Komponenten werden jetzt oberhalb des Standard-Contents platziert.

    Tip:

    Bei Seitenvarianten, die aus einem Stamm-Management-Kontext freigegeben wurden (siehe Freigaben von Content-Elementen), ist die Option, eigene Komponenten über dem Standard-Content anzuzeigen, nicht verfügbar.

Ererbte Parameter bearbeiten

Es werden standardmäßig alle Parameterwerte aus den Stammobjekten vererbt. Jedoch können sie auf jeder Vererbungs-Ebene überladen sein. Überladene Werte können auf den ursprünglichen (ererbten) Wert entsprechend zurückgesetzt werden.

Ererbte Parameter Überladen

Um einen ererbten Parameterwert zu überladen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Klicken Sie auf den Namen der Seitenvariante, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Beginnen Sie im Abschnitt Parameter mit der Bearbeitung der gewünschten Parameterwerte.

    Wird ein Wert bearbeitet, so zeigt ein Stift-Symbol () neben dem Wert an, dass dieser Wert nicht mehr ererbt, sondern überladen ist.

  4. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

    Anderenfalls klicken Sie auf Zurücksetzen, um die Änderungen zu verwerfen.

Überladen von Parametern Zurücksetzen

Um einen überladenen Parameter auf seine ursprünglichen (ererbten) Wert zurückzusetzen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Klicken Sie auf den Namen der Seitenvariante, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Klicken Sie im Abschnitt Parameter auf Zurücksetzen auf geerbte Werte.

    Es werden Kontrollkästchen vor den bearbeiteten Parametern angezeigt.

  4. Wählen Sie die Parameter, die zurückgesetzt werden sollen.

    Verwenden Sie die Kontrollkästchen, um die gewünschten Parameter auszuwählen.

  5. Klicken Sie auf Zurücksetzen.

    Die ausgewählten Parameter werden auf ihre ursprünglichen (ererbten) Werte zurückgesetzt.

Zugewiesene Objekte verwalten

Seitenvarianten können verschiedenen Business-Objekten zugewiesen werden. Das beinhaltet:

  • Kundensegmente

    Sie können mehrere Kundensegmente hinzufügen. Wenn Sie das tun, können Sie keine Aktionen oder Kampagnen zuordnen.

  • Aktionen

    Sie können eine Aktion zuordnen. Wenn Sie das tun, können Sie keine Kundensegmente hinzufügen oder Kampagnen zuordnen.

  • Kampagnen

    Sie können eine Kampagne zuordnen. Wenn Sie das tun, können Sie keine Kundensegmente hinzufügen oder Aktionen zuordnen.

  • A/B-Tests

    Sie können eine oder mehrere Testgruppen aus (mehreren) A/B-Tests hinzufügen. Davon werden die anderen Business-Objekt-Beziehungen nicht beeinträchtigt.

Erzeugen von Business-Objekt-Beziehungen

Um Business-Objekt-Beziehungen für Seitenvarianten zu erzeugen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Klicken Sie auf den Namen der Seitenvariante, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Wechseln Sie zum Register Eigenschaften.

    Das Register Eigenschaften sieht u.a. vor, Business-Objekt-Beziehungen zu verwalten.

  4. Die gewünschten Business-Objekte hinzufügen bzw. zuordnen.

    Klicken Sie Hinzufügen/Zuordnen, um einen Dialog zu öffnen und die gewünschten Business-Objekte auszuwählen und hinzuzufügen.

  5. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

    Anderenfalls klicken Sie auf Zurücksetzen, um die Änderungen zu verwerfen.

Entfernen von Objekt-Beziehungen

Um Business-Objekt-Beziehungen für Seitenvarianten zu entfernen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Klicken Sie auf den Namen der Seitenvariante, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Wechseln Sie zum Register Eigenschaften.

    Das Register Eigenschaften sieht u.a. vor, Business-Objekt-Beziehungen zu verwalten.

  4. Entfernen Sie Business-Objekt-Beziehungen wie erforderlich.

    Um eine Zuordnung zu einem Kundensegment oder einer A/B-Testgruppe zu entfernen, wählen Sie die entsprechende Zuordnung und klicken Sie Entfernen.

    Um eine Zuordnung zu einer Aktion oder Kampagne zu entfernen, klicken Sie neben der entsprechenden Zuordnung auf Aufheben.

  5. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

    Anderenfalls klicken Sie auf Zurücksetzen, um die Änderungen zu verwerfen.

Seitenvarianten kopieren

Beim Kopieren von Seitenvarianten werden alle dazugehörigen Content-Elemente und die zugeordneten Komponenten ebenfalls kopiert. Um eine Seitenvariaten zu kopieren:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Klicken Sie auf das Symbol in der Zeile der Seitenvariante, die Sie kopieren wollen.

    Der Dialog Seitenvariante kopieren wird angezeigt.

  3. Bearbeiten Sie Namen, ID und Publikationszeitraum wie erforderlich.

  4. Klicken Sie Übernehmen.

    Die Kopie der Seitenvariante wird angelegt und in der Detailansicht geöffnet.

Komponenten zu Seite hinzufügen

Das Register Content der Detailansicht ermöglicht das explizite Zuordnen einer Komponente zu einem Bereich der Seitenvariante.

Hinweis:

Nur publizierte Komponenten können online sichtbar sein, unabhängig von ihren Seitenbereich-Zuordnungen.

Damit Komponenten in der Storefront sichtbar werden, müssen die Publikationsdauer sowie der Sichtbarkeitszeitraum und die Priorität übereinstimmen.

Bestehende Komponenten zuordnen

Um eine bestehende Komponente in eine Seitenvariante einzufügen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Klicken Sie auf den Namen der Seitenvariante, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Klicken Sie im Abschnitt des gewünschten Seitenbereichs auf Zuordnen.

    Der Dialog Komponente zuordnen wird angezeigt. Unterhalb des Abschnitts Priorität und Sichtbarkeit listet der Dialog die verfügbaren Komponenten auf. Falls erforderlich, können Sie die Liste durch Angabe eines Komponentennamens oder -typs eingrenzen.

  4. Legen Sie den Sichtbarkeitszeitraum und die Priorität fest.

  5. Wählen Sie die Komponten aus, die dem Seitenbereich zugeordnet werden sollen.

  6. Klicken Sie auf Zuordnen.

    Die Komponenten werden dem Seitenbereich der aktuellen Seitenvariante zugeordnet.

Neue Komponente hinzufügen

Um eine neue Komponente anzulegen und sie direkt einem Seitenbereich der Seitenvariante zuzuordnen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Klicken Sie auf den Namen der Seitenvariante, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Klicken Sie im Abschnitt des gewünschten Seitenbereichs auf Neu.

    Der Dialog Neue Komponente wird angezeigt.

  4. Bearbeiten Sie die Details der Komponente wie erforderlich.

    Geben Sie einen Namen an, bearbeiten Sie die ID, falls erforderlich, wählen Sie ein Komponenten-Template aus, definieren Sie den Publikationszeitraum und legen Sie den Sichtbarkeitszeitraum und die Priorität fest.

  5. Klicken Sie Übernehmen.

    Die neue Komponente wird angelegt und in der Detailansicht zur Bearbeitung geöffnet, siehe Komponenten bearbeiten.

    Um zur Detailansicht der Seitenvariante zurückzukehren, klicken Sie auf Zurück zur Liste.

Komponentenpriorität festlegen

Falls einem Seitenbereich mehrere Komponenten mit überlappenden Sichtbarkeitszeiträumen zugeordnet sind, bestimmt die Priorität die Reihenfolge, in der die online gesetzten Komponenten tatsächlich angezeigt wird. Um die Priorität von zugeordneten Komponenten festzulegen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Klicken Sie auf den Namen der Seitenvariante, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Benutzen Sie im Abschnitt Content die Pfeile in der Spalte Priorität, um die Priorität der Komponenten zu ändern.

    Durch Klick auf eine Pfeil-Schaltfläche wird die entsprechende Komponente nach oben oder unten verschoben, wodurch unmittelbar die Anzeigepriorität geändert wird.

Komponenten-Sichtbarkeit festlegen

Die Sichtbarkeit bezieht sich auf die Verfügbarkeit der Komponente in einem bestimmten Kontext, d.h. innerhalb eines Seitenbereichs, wenn sie explizit zugeordnet wurde. Die Sichtbarkeit legt die Gültigkeitsdauer einer Seitenbereichs-Komponenten-Zuordnung fest. Falls einem Seitenbereich mehrere Komponenten mit überlappenden Sichtbarkeitszeiträumen zugeordnet sind, bestimmt die Priorität die Reihenfolge, in der die online gesetzten Komponenten tatsächlich angezeigt wird. Damit Komponenten in der Storefront sichtbar werden, müssen die Publikationsdauer sowie der Sichtbarkeitszeitraum und die Priorität übereinstimmen.

Um die Sichtbarkeit von zugeordneten Komponenten festzulegen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Klicken Sie auf den Namen der Seitenvariante, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Klicken Sie auf den Sichtbarkeitszeitraum der entsprechenden Komponente.

    Das Dialogfeld Priorität und Sichtbarkeit wird geöffnet.

  4. Bearbeiten Sie den Sichtbarkeitszeitraum und die Priorität wie gewünscht.

  5. Klicken Sie auf OK, um Änderungen zu speichern.

    Anderenfalls klicken Sie auf Abbrechen, um die Änderungen zu verwerfen.

Zuordnungen von Komponenten zu Seitenvarianten aufheben

Um die Zuordnung einer Komponente zu einer Seitenvariante aufzuheben:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Klicken Sie auf den Namen der Seitenvariante, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Markieren Sie Zuordnung, die Sie aufheben wollen.

    Markieren Sie im Abschnitt Content das Kontrollkästchen der entsprechenden Komponentenzuordnung.

  4. Klicken Sie auf Aufheben und bestätigen Sie die Operation.

    Die Zuordnung der Komponente zur aktuellen Seitenvariante wird entfernt.

Seitenvarianten-Zuordnungen verwalten

Das Verwalten von Seitenvarianten-Zuordnungen besteht aus dem Hinzufügen von Seitenvarianten zu Seiten (Content/Systemseiten, Kategorieseiten oder Produktseiten, wie vom Seitenvariantentyp zulässig) und dem Entfernen dieser Zuordnungen.

Seitenvarianten zu Seiten zuordnen

Es gibt zwei Arten, um eine bestehende Seitenvariante einer bestehenden Seite zuzuordnen: Entweder beginnen Sie, wie unten beschrieben, bei der Seitenvariante über das Register Zuordnungen, oder Sie beginnen bei der Seite über das Register Seitenvarianten, siehe Seitenvarianten zu Seite hinzufügen.

Um eine Seiten-Zuordnung für eine Seitenvariante zu erstellen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Klicken Sie auf den Namen der Seitenvariante, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Öffnen Sie das Register Zuordnungen.

    Für jeden Seitentyp gibt es einen Abschnitt. Abschnitte mit vorhandenen Zuordnungen werden aufgeklappt, andere zugeklappt.

    Hinweis:

    Die verfügbaren Seiten hängen von den zulässigen Zuordnungen ab, je nach Seitenvariante.

  4. Klicken Sie im entsprechenden Abschnitt auf Zuordnen.

    Der Assistent zum Zuordnen der aktuellen Seitenvariante zu einer bestehenden Seite des gewählten Typs wird geöffnet.

  5. Markieren Sie die gewünschte Seite und klicken Sie auf Weiter.

  6. Legen Sie den Sichtbarkeitszeitraum und die Priorität fest und klicken Sie auf Fertigstellen.

    Die Seitenvariante wird der gewählten Seite zugeordnet.

Zuordnungen von Seitenvarianten aufheben

Um die Zuordnung einer Seitenvariante zu einer Seite aufzuheben:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Klicken Sie auf den Namen der Seitenvariante, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Öffnen Sie das Register Zuordnungen.

  4. Markieren Sie Zuordnung, die Sie aufheben wollen.

    Markieren Sie das Kontrollkästchen der gewählten Zuordnung im entsprechenden Abschnitt.

  5. Klicken Sie auf Aufheben und bestätigen Sie die Operation.

    Die Zuordnung der Seitenvariante zur aktuellen Seite wird entfernt.

Publikationseinstellungen für Seitenvarianten festlegen

Publikation bezieht sich auf die allgemeine Verfügbarkeit der Seitenvariante. Nur publizierte Seitenvarianten können sichtbar sein, d.h. online sein. Um den Publikationszeitraum für eine Seitenvariante festzulegen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Klicken Sie auf das Publikationsdatum der Seitenvariante, die Sie bearbeiten wollen.

    Das Register Eigenschaften der Detailansicht wird geöffnet.

  3. Legen Sie Publikations-Status und -Dauer fest wie erforderlich.

    Markieren Sie das Optionsfeld des gewünschten Status und geben Sie die Publiziert-von- und Publiziert-bis-Daten und -Zeiten an.

    Hinweis:

    Leeres Enddatum und -zeit setzen die Publikationsdauer auf "unbegrenzt".

  4. Klicken Sie Übernehmen.

    Die Einstellungen für Publikations-Status und -Dauer werden sofort aktiv.

Seitenvarianten-Label verwalten

Label ermöglichen die Definition von Gruppen abhängiger Objekte, um dann bestimmte Aktionen mit diesen Objekten auszuführen. Informationen zum Erzeugen von Labeln, siehe Label anlegen. Für weitere Informationen zur Ausführung label-basierter Aktionen, siehe Aktionen für Objekte mit Labeln durchführen.

Label zu Seitenvarianten hinzufügen

Um einer Seitenvariante ein Label zuzuordnen:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Klicken Sie auf den Namen der Seitenvariante, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Öffnen Sie das Register Label.

    Eine Liste der Label, die dieser Seitenvariante zugeordnet sind, wird angezeigt.

  4. Klicken Sie auf Neu, um eine neue Label-Zuordnung zu definieren.

    Eine Liste der verfügbaren Label wird angezeigt.

  5. Markieren Sie das gewünschte Label und klicken Sie auf OK.

    Das ausgewählte Label wird der aktuellen Seitenvariante zugeordnet.

Label von Seitenvarianten entfernen

Um die Zuordnung eines Labels zu einer Seitenvariante aufzuheben:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Klicken Sie auf den Namen der Seitenvariante, die Sie bearbeiten wollen.

    Das Register Content der Detailansicht wird geöffnet.

  3. Öffnen Sie das Register Label.

    Eine Liste der Label, die dieser Seitenvariante zugeordnet sind, wird angezeigt.

  4. Markieren Sie das Kontrollkästchen der Labelzuordnung, die Sie aufheben wollen.

  5. Klicken Sie auf Aufheben und bestätigen Sie die Operation.

    Die Zuordnung des gewählten Labels zur aktuellen Seitenvariante wird entfernt.

Seitenvarianten in Stapelprozessen bearbeiten

Bestimmte Operationen in der Verwaltung von Seitenvarianten können mittels Stapelprozessen mit mehreren Seitenvarianten gleichzeitig ausgeführt werden. Operationen, die als Stapelprozesse ausgeführt werden können, sind:

  • Publikationszeitraum der ausgewählten Seitenvarianten festlegen

  • Ausgewählte Seitenvariante Seiten zuordnen

  • Eine Komponente zu den ausgewählten Seitenvarianten zuordnen

  • Ein Label zu den ausgewählten Seitenvarianten zuordnen

  • Markierte Seitenvarianten löschen

Weitere Informationen zum Bearbeiten von Seitenvarianten per Stapelprozess, siehe Objekte in Stapelprozessen bearbeiten.

Seitenvarianten publizieren

In einer Umgebung der Datenreplizierung müssen Seitenvarianten, die im Bearbeitungssystem erzeugt werden, im Zielsystem veröffentlicht werden. Dies geschieht entweder automatisch im Kontext von regulärer Datenreplizierung (siehe Verwaltung von Replikationsaufgaben) oder manuell, wie unten beschrieben.

Um Seitenvarianten manuell zu publizieren, gehen Sie wie folgt vor:

  1. Wählen Sie den Management-Kontext aus der Kontextauswahlbox, wählen Sie dann Content | Seiten-Varianten.

    Die Listenansicht der Seitenvarianten wird angezeigt.

  2. Markieren Sie die Kontrollkästchen der zu publizierenden Seitenvarianten in der Listenansicht.

    Alternativ können Sie eine Suche ausführen und auf Alles auswählen klicken, um das Suchergebnis auszuwählen.

  3. Klicken Sie Publizieren im Zielsystem.

    Die Liste konfigurierter Zielsysteme wird angezeigt.

  4. Wählen Sie das gewünschte Zielsystem und die Lösung für einen Sperrkonflikt.

    Für Informationen zu den Sperrkonflikt-Lösungen, siehe Objektsperren.

  5. Klicken Sie auf Fertigstellen.

    Die gewählten Seitenvarianten werden an das gewählte Zielsystem repliziert.
