Vertriebsorganisationen und Channels werden über individuelle Websites repräsentiert, d. h. ihre jeweilige Geschäftsapplikation (wie z. B. Commerce Management für Vertriebsorganisationen und Channels). Diese Applikationen (oder Sites) bilden die vorgesehenen Geschäftsprozesse ab und stellen die für administrative und operative Aufgaben notwendige Arbeitsoberfläche bereit.

Die Organisationsverwaltung bietet Zugriff auf Funktionen zur Steuerung verschiedener site-spezifischer Einstellungen, einschließlich

  • Servergruppen-Zuordnung

  • HTTPS-Verwendung

  • URL-Manipulation

  • Online-Status

Die Site-Einstellungen in der Organisationsverwaltung entsprechen den allgemeinen Site-Einstellungen, die über die System-Management gesteuert werden.

Hinweis:

Die Organisationsverwaltung zeigt alle Applikationen der Vertriebsorganisation an. Sie können hier jedoch Einstellungen für channel-spezifische Applikationen, die von Administratoren der Vertriebsorganisation angelegt wurden, nicht ändern.

Servergruppen

In einer Intershop Commerce Management Umgebung mit mehreren Applikationsservern können Sie einzelne Applikationsserver bestimmten Servergruppen zuweisen; in einer Standardinstalltion z. B.: WFS für Web-Frontend-Abfragen oder BOS für Backoffice-Abfragen.

Um Anfragen an die Web-Applikation weiterzuleiten, können in der Organisationsverwaltung einzelne Sites (Applikationen) bestimmten Servergruppen zugewiesen werden.

Um eine Applikation einer Servergruppe zuzuweisen:

  1. Klicken Sie in der Navigationsleiste auf Site-Management.

    Eine Liste der verfügbaren Sites wird angezeigt.

  2. Klicken Sie auf den Namen der Site, die Sie bearbeiten wollen.

    Die Seite Site-Einstellungen wird angezeigt.

  3. Geben Sie eine Servergruppe im entsprechenden Feld an.

    Hinweis:

    Stellen Sie sicher, dass die Servergruppe im Web-Adapter definiert und mindestens ein Applikationsserver dieser Servergruppe zugewiesen ist.

  4. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

    Anderenfalls klicken Sie auf Zurücksetzen, um die Änderungen zu verwerfen.

Informationen über das Konfigurieren von Servergruppen im Web-Adapter und über das Zuweisen von Applikationsservern zu Servergruppen finden Sie im Dokument Guide - Web Adapter Settings in der Intershop Knowledge Base.

Server-Konfiguration mit ausschließlich gesicherten URLs

Die Intershop Commerce Management Applikation kann sowohl abgesicherte als auch ungesicherte URLs benutzen. Web-Applikationen können abgesicherte URLs verwenden, um einen gute Suchindex-Position zu erlangen.

Die Intershop Commerce Management Applikation kann so konfiguriert werden, dass ausschließlich abgesicherte URLs erzeugt und ungesicherte Anfragen entsprechend umgeleitet werden.

Um die Funktion zu aktivieren/deaktivieren, muss die Eigenschaft SecureAccessOnly konfiguriert werden. Folgende Einstellungen stehen zur Verfügung:

Tabelle 5: Einstellmöglichkeiten für SecureAccessOnly
Einstellung Beschreibung
true Ausschließlich abgesicherte URLs werden erzeugt. Ungesicherte Anfragen werden entsprechend umgeleitet.
false Der abgesicherte Modus wird deaktiviert. Sowohl abgesicherte als auch ungesicherte URL können erzeugt werden.

Die Eigenschaft SecureAccessOnly kann eingestellt werden für Sites, Applikationen oder für ganze Cluster (vollständiges System). Speicherorte der entsprechenden Konfigurationsdateien sind:

  • Applikations-spezifisch:

    $IS_SHARE/system/config/apps/<appName>/ssl.properties

  • Site-spezifisch:

    $IS_SHARE/system/config/domains/<domainName>/ssl.properties

  • Systemweite Eigenschaft:

    $IS_SHARE/system/config/cluster/ssl.properties

Sicherer Online-Zugriff mit HTTPS

In der Organisationsverwaltung können Sie den Online-Zugriff auf Ihre Web-Applikationen durch die Verwendung des Protokolls HTTPS absichern.

Hinweis:

Um diese Funktion zu nutzen, muss die HTTPS-Unterstützung im Webserver und im Web-Adapter aktiviert sein. Weitere Informationen zu diesem Thema finden Sie in Guide - Web Adapter Settings und Guide - Web Server Settings in der Intershop Knowledge Base.

Um HTTPS für Sites zu aktivieren:

  1. Klicken Sie in der Navigationsleiste auf Site-Management.

    Eine Liste der verfügbaren Sites wird angezeigt.

  2. Klicken Sie auf den Namen der Site, die Sie bearbeiten wollen.

    Die Seite Site-Einstellungen wird angezeigt.

  3. Markieren Sie das Kontrollkästchen HTTPS.

  4. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

    Anderenfalls klicken Sie auf Zurücksetzen, um die Änderungen zu verwerfen.

Suchmaschinenoptimierung: URL-Manipulation

Intershop Commerce Management bietet eine Funktion zur URL-Manipulation, mit deren Hilfe die Web-Applikationen suchmaschinenfreundliche URLs erzeugen können. Damit ist die URL-Manipulation eine von mehreren Möglichkeiten, Intershop Commerce Management-Sites für Web-Suchmaschinen zu optimieren.

Die Standardimplementierung der URL-Manipulation in Intershop Commerce Management benutzt cartridge-spezifische Konfigurationsdateien zum Festlegen der Regeln zum Umformen langer Intershop Commerce Management-URLs in suchmaschinenfreundliche URLs. Diese Dateien, urlrewrite.properties, werden für jede betroffene Cartridge abgespeichert unter <IS.INSTANCE.SHARE>/system/cartridges/<cartridge_name>/release/urlrewrite. Zum Bearbeiten der Regeln sind Kenntnisse regulärer Ausdrücke erforderlich.

Die Eigenschaften intershop.urlrewrite.CheckSource und intershop.urlrewrite.CheckSourceInterval , Bestandteil der Datei appserver.properties , steuern den Status und das Aktualisierungsverhalten für die URL-Manipulation. Weitere Informationen finden Sie im Guide - Intershop Application Server Settings in der Intershop Knowledge Base.

Um die URL-Manipulation für Applikationen zu aktivieren:

Achtung:

Stellen Sie sicher, dass die Ersetzungsregeln in den Datei(en) urlrewrite.properties korrekt konfiguriert sind, bevor Sie die URL-Manipulation aktivieren.

  1. Klicken Sie in der Navigationsleiste auf Site-Management.

    Eine Liste der verfügbaren Sites wird angezeigt.

  2. Klicken Sie auf den Namen der Site, die Sie bearbeiten wollen.

    Die Seite Site-Einstellungen wird angezeigt.

  3. Markieren Sie das Kontrollkästchen URL-Manipulation.

  4. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

    Anderenfalls klicken Sie auf Zurücksetzen, um die Änderungen zu verwerfen.

Tip:

Die meisten Suchmaschinen verschlechtern die Patzierung von Sites mit ungesicherten URLs. Web-Applikationen können abgesicherte URLs verwenden, um einen gute Suchindex-Position zu erlangen. Für weitere Informationen, siehe Server-Konfiguration mit ausschließlich gesicherten URLs.

Online-Status

In der Organisationsverwaltung können Sie den Online-Status einer Site festlegen. Um den Online-Status zu ändern:

  1. Klicken Sie in der Navigationsleiste auf Site-Management.

    Eine Liste der verfügbaren Sites wird angezeigt.

  2. Klicken Sie auf den Namen der Site, die Sie bearbeiten wollen.

    Die Seite Applikationseinstellungen wird angezeigt.

  3. Wählen Sie den gewünschten Status in der Auswahlbox aus.

    Folgende Status stehen zur Verfügung:

    Tabelle 6: Site-Status
    Status Beschreibung
    Live Zugriff auf die Site ist möglich.
    Wartung Die Site ist nur für Commerce Management-Benutzer zugänglich. Webfront-/Shop-Anfragen an die Site werden blockiert.
    Deaktiviert Die Site ist nur für Administratoren im System-Management oder in der Organisationsverwaltung zugänglich. Commerce Management-Anfragen und Webfront-/Shop-Anfragen an die Site werden blockiert.

  4. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

    Anderenfalls klicken Sie auf Zurücksetzen, um die Änderungen zu verwerfen.
